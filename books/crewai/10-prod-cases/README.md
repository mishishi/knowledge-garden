写技术文章最怕「教学 demo 跟生产差十万八千里」。这一章调研公开声明过用 CrewAI 的公司，看他们在生产里**实际**怎么用、解决了什么、踩过什么坑。不是营销 PR 复述，是拆解。

## 每个 case 的拆解维度

不是堆公司名+logo，每个 case 都按这 5 个维度拆：

1. **业务问题**：他们在解决什么（具体到 ROI 数字，能找到的话）
2. **Crew 拓扑**：用几个 Agent、Process 怎么选、Flow 还是纯 Crew
3. **关键技术选型**：v0.x 还是 v1.x、Knowledge 还是 Tool、Memory 怎么配
4. **踩过的坑**：生产里翻车的具体案例（不是「我们也遇到了挑战」这种空话）
5. **今天的状态**：还在用吗？迁走了吗？为什么？
