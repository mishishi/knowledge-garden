# 09. 家庭服务机器人：消费级具身智能

家庭服务机器人是 embodied agent **离消费者最近**的应用——也是商业化最难的方向。2026 年开始有真实家庭试点（1X Neo、Figure 02、Tesla Optimus）。

这一章讲家庭服务机器人的核心挑战、2026 年现状、典型任务、未来 3 年展望。

## 为什么家庭服务机器人最难

**工业机器人（成熟）**：环境可控、任务固定、容错要求低 → 已经普及。

**家庭服务机器人（早期）**：环境千变万化、任务开放、容错要求高 → 2026 年才开始真实家庭试点。

**5 个核心挑战**：

**1. 环境不可预测**

家里随时可能变——小孩乱丢玩具、宠物乱跑、物品位置每天都不同。

**2. 任务开放**

工业机器人做"把 A 放到 B"，家庭机器人做"帮我收拾桌子"——任务定义本身就不清楚。

**3. 安全要求极高**

家里有老人小孩，机器人不能撞人、不能夹手、不能摔东西。

**4. 成本敏感**

工业机器人 50 万人民币 OK，家庭机器人 5 万人民币都嫌贵。

**5. 用户期望高**

人型机器人天然引发高期望——大家期望它像科幻电影一样能干，但实际上离那个水平差 5-10 年。

## 2026 年家庭服务机器人现状

**产品**：1X Neo、Figure 02、Tesla Optimus、宇树 H1（家用版）、Sanctuary Phoenix。

**当前能力**（2026 年中）：

- 自主导航（避障、找路）→ **98% 成功率**
- 简单抓取（已知物体、固定位置）→ **85% 成功率**
- 简单对话（"帮我拿水") → **75% 成功率**
- 长 horizon 任务（"做饭"）→ **30% 成功率**
- 复杂操作（"叠衣服") → **10% 成功率**

**真实部署**：2026 年中全球大约 5 万台家庭服务机器人在真实家庭运行（vs 100 万台工业机器人）。

**价格区间**：5-20 万人民币，租赁模式 2000-5000 元/月。

## 典型家庭任务分类

### Level 1：简单任务（已可用）

- 巡逻 / 安防
- 视频通话
- 拉窗帘
- 开关灯
- 递物品（用户递位置，机器人去拿）
- 提醒（吃药、定时）

### Level 2：中等任务（2026-2027 可用）

- 扫地 + 拖地（已有 iRobot / 科沃斯，但加 embodied 智能后能力更强）
- 简单整理（把桌面物品归位）
- 简单烹饪（按菜谱加热预制菜）
- 老人看护（监测异常、呼叫）

### Level 3：复杂任务（2027-2030）

- 完整做饭（从备菜到出锅）
- 整理房间（叠衣服、收拾散落物品）
- 照顾老人（搀扶、辅助如厕）
- 照顾小孩（陪伴、教育）

### Level 4：高级任务（2030+）

- 完全自主管家
- 多任务编排（同时处理多项家务）
- 情感交互（识别情绪、安慰）

## 关键技术栈

家庭服务机器人需要 6 大能力叠加：

**1. 感知（Perception）**

- 视觉（物体识别、人物识别、姿态估计）
- 听觉（语音识别、声源定位）
- 触觉（抓取力反馈、防撞）
- proprioception（自身关节状态）

**2. 导航（Navigation）**

- SLAM 建图
- 路径规划
- 动态避障

**3. 操作（Manipulation）**

- 抓取（已知 + 未知物体）
- 操作（开门、倒水、拧瓶盖）
- 双手协作

**4. 对话（Interaction）**

- 语音 ASR / TTS
- 自然语言理解
- 多轮对话 + 上下文

**5. 任务规划（Task Planning）**

- 高层任务拆解（LLM / VLM）
- 子任务调度
- 异常处理

**6. 安全（Safety）**

- 碰撞检测
- 关节限位
- 异常停机
- 人在回路（human-in-the-loop）

## 关键挑战的工程解法

**挑战 1：环境不可预测**

**解法**：持续学习 + 在线适应。机器人每天运行都在学新东西，越用越聪明。

```python
class ContinualLearningAgent:
    async def daily_learning_loop(self):
        while True:
            # 收集当天运行数据
            episodes = await self.collect_today_episodes()
            
            # 离线训练（夜间）
            await self.finetune(episodes)
            
            # 验证新模型
            new_performance = await self.validate()
            
            # 如果新模型更好，部署
            if new_performance > current_performance * 1.02:
                await self.deploy_new_model()
            
            await asyncio.sleep(86400)  # 每天跑一次
```

**挑战 2：任务开放**

**解法**：LLM / VLM 做任务规划，把开放指令翻译成具体动作。

**挑战 3：安全要求高**

**解法**：多层安全防护。

```python
class SafetyLayer:
    async def check(self, action):
        # 1. 碰撞检测（基于 SLAM）
        if await self.will_collide(action):
            return False, "will_collide"
        
        # 2. 关节限位
        if await self.exceeds_joint_limits(action):
            return False, "joint_limit"
        
        # 3. 力矩限位
        if await self.exceeds_torque(action):
            return False, "torque_limit"
        
        # 4. 速度限位（家庭里慢速优先）
        if await self.exceeds_safe_speed(action):
            return False, "speed_limit"
        
        return True, "ok"
```

**挑战 4：成本敏感**

**解法**：模块化设计 + 租赁模式。

```python
# 模块化：用户可以只买基础版，加功能付费
class ModularHomeRobot:
    def __init__(self, modules):
        self.modules = modules  # 用户买的模块
    
    def available_skills(self):
        return [m.skill for m in self.modules if m.enabled]
```

## 未来 3 年展望

**2026 年**：家庭试点 5-10 万台，主要在高端家庭。

**2027 年**：规模 50-100 万台，价格降到 5 万人民币以内，能力提升到 Level 2。

**2028 年**：规模 500 万台，进入中产家庭，能力提升到 Level 3 边界。

**2030 年**：规模 5000 万台+，部分发达国家普及。

**关键不确定性**：

- VLA 模型能不能突破到 Level 3 能力
- 硬件成本能不能降到消费级
- 法规（特别是医疗 / 老人看护）能不能跟上
- 公众接受度

## 实战建议

**想做家庭机器人创业**：先做工业 / 仓储（已成熟），盈利后再做家庭。

**想做家庭机器人研究**：从单一任务（递物品、扫地）切入，验证后扩展。

**想买家庭机器人**：现在（2026）买要谨慎——能力有限、价格贵、维护麻烦。等 2027-2028 年更成熟。
