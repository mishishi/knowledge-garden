# 04. 用 AI 设计产品

我之前有个偏见 — "AI 没法做设计"。

偏见来源是我自己用过几次 AI 生成 UI，出来的东西都"看着像 AI"。**配色像 2020 年的 Material Design，间距像 Bootstrap 模板，按钮像 SaaS 烂大街的紫色**。

2025 年我把这个偏见彻底推翻了。原因：我用 AI 重新设计了 side project 的 landing page，2 小时内出了 12 个版本，我自己用 3 周都不一定做得到这个量。**前提是 — 我懂基本的设计判断，知道哪个版本能用。**

这一章讲怎么用 AI 做产品设计。**不教你怎么成为设计师**，教你怎么用 AI 当你的"第一个设计师" + 怎么在 4 周后判断是否要请真人设计师。

## AI 做设计的 4 种用法（按能力排序）

不是所有 AI 都能做设计。我把这两年试过的工具按能力排：

| 工具 | 能力 | 价格 | 适合 |
|---|---|---|---|
| **v0.dev**（Vercel） | 文字 → Tailwind UI 组件 | 免费够用 | 落地页 / 简单 web app |
| **Galileo AI** | 文字 → 完整 UI 草图 | 免费层 | 早期 mockup |
| **Figma AI** | 已有设计 → 自动改 | 付费 | 已有 Figma 流程的人 |
| **Relume** | 文字 → 完整 web 组件 + 内容 | 付费 | 营销页 / SaaS dashboard |
| **Midjourney / DALL-E 3** | 文字 → 图片 | 付费 | 营销图 / 概念图（非 UI） |

**重点是 — 这些工具 5 年前不存在。** v0.dev 2023 年 11 月才上线。Galileo AI 2024 年 2 月。现在它们每周都更新。

我自己的用法是 **"v0 出第一版 → Figma 调细节 → 真人设计师优化"**。前两步都靠 AI，第三步不到 30% 的项目需要。

## 用 v0.dev 30 分钟做落地页

我自己的 side project 80% 的落地页是 v0.dev 做的。具体流程：

1. 打开 v0.dev，输入"我想做一个 [产品] 的 landing page，目标用户是 [X]，主要卖点 [Y]"
2. 等 30 秒，AI 给我 4 个版本
3. 选最接近的一个，继续聊细节（"主标题换成 [这个] / 配色换成 [这个] / 加上一个 [这个 section] / 把 hero 图换成 [这个风格]"）
4. 重复 3-5 轮，直到满意
5. 复制代码到自己的项目，跑起来

**完整时间 30-60 分钟。** 我自己做要 3-5 天。

**v0.dev 的真实 prompt 模板：**

```
我想做一个 [产品名] 的 landing page。

产品：
- 一句话定义：[X 是给 Y 用的 Z 工具]
- 目标用户：[具体画像，含年龄 / 职业 / 痛点]
- 3 个核心卖点（用户最想要的 3 件事）：
  1. [具体好处 1，要可衡量，比如"省 5 小时/周"]
  2. [具体好处 2]
  3. [具体好处 3]
- 视觉风格：[参考哪个品牌 / 极简 / 鲜艳 / 专业 / 友好]
- 必有的 section：
  - Hero（大标题 + 副标题 + 一个 CTA 按钮）
  - 3 个核心卖点（每点配一个 emoji 图标 + 简短描述）
  - 用户证言（3 段引言 + 头像 + 名字 + 职位）
  - 定价（3 档：免费 / 个人 / 团队）
  - 底部 CTA
  - FAQ（5 个最常见问题）

不要：
- 不要 hero 图占满全屏（首屏加载慢）
- 不要用 3D 效果 / 动画 / 视频背景
- 不要用 emoji 在文案里（用图标）
```

**v0.dev 出来的版本长这样（我最近一个项目实际跑的）：**

- **首屏标题**："**Stop wasting 5 hours/week on SEO descriptions**"
- **副标题**："AI writes your real estate listing SEO in 60 seconds. Trained on 50,000 high-ranking listings."
- **CTA 按钮**：紫色 "Get your first 10 free" （不是 "Sign up"，是 "Get your first 10 free" — 具体价值 + 免费）
- **3 个卖点**：每个 30 字以内 + 一个 SVG 图标
- **证言**：3 个房产中介头像 + 引言 + 城市 / 公司

这种落地页 5 年前需要设计师 + copywriter + 1 周时间。**现在 30 分钟。**

## 用 Galileo AI 做产品 mockup

v0.dev 适合落地页（web）。**App 设计**用 Galileo AI 更合适。

Galileo 的能力 — 给你文字描述，AI 生成 4-6 个完整 app 屏幕的设计图（含 navigation、components、interaction states）。

**真实 prompt：**

```
我做一个 [app 类型]，目标用户是 [画像]。

请生成 5 个核心 screen：

1. **首页（home）**
   - 顶部 nav（logo + 3 个 menu + 头像）
   - Hero section（今天最重要的内容）
   - 3 个推荐卡片
   - 底部 tab bar（4 个 tab：home / search / add / profile）

2. **搜索结果页**
   - 顶部搜索框（带返回箭头）
   - 筛选器 chip（3 个常见 + 1 个"全部"）
   - 列表卡片（thumbnail + 标题 + 2 行描述 + 1 个 action）

3. **详情页**
   - 顶部（返回 + share + 收藏）
   - 大封面图
   - 标题 + 作者 + 1 行 metadata
   - 主体内容（4-5 段文字）
   - 底部 sticky CTA

4. **创建 / 编辑页**
   - 顶部（取消 + 保存）
   - 主输入区（大文本框）
   - 辅助输入（2-3 个选项）
   - 底部一个主要按钮

5. **个人页**
   - 顶部 banner（头像 + 名字 + 1 行 bio）
   - 4 个 stat（数字 + 标签）
   - 内容 tab（2-3 个）
   - 内容列表

风格要求：
- 干净、留白多、字号大
- 主色 [具体颜色，比如 #FF5733]
- 字体 [具体字体，比如 Inter]
- 风格参考 [具体 app，比如 Linear / Notion / 微信读书]
```

我用这套 prompt 做过 3 个 app 的 mockup，每个花 20-30 分钟。**比用 Figma 重新画快 10 倍**，而且能给"非设计师"提供"视觉的判断依据"。

**关键 insight：mockup 不是最终设计，是"让团队 / 投资人 / 早期用户能看懂你要做什么"的工具。** 用 AI 出来后你可以拿这个去 Figma 找真人设计师"按这个改"。

## 不需要设计师的 5 个边界

AI 设计的边界很明确。**5 种情况你必须请真人设计师**：

1. **复杂数据可视化**（dashboard、图表、报告）— AI 画 dashboard 还是不好看，真人设计师懂信息层级
2. **品牌识别系统**（logo、字体、配色全套）— AI 出 logo 一眼像 AI，真人设计师懂"为什么这个 logo 看起来对"
3. **打印 / 实物**（名片、t-shirt、贴纸）— AI 不知道物理世界的约束
4. **复杂交互动效**（拖拽、动画、状态切换）— AI 出的动效能用但不优雅
5. **国际化 + 跨文化**（中文 + 英文 + 阿拉伯文 RTL 适配）— AI 经常搞错

我自己 side project 的 4 次设计师合作经历：
- 第 1 次：AI 出方案，设计师"按这个优化"，**2 周 $2000**
- 第 2 次：AI 出方案，我自己改，**0 时间 $0**
- 第 3 次：AI 出方案，设计师"只改配色和字体"，**3 天 $500**
- 第 4 次：AI 出的直接上线，**0 时间 $0**

经验：**等你验证 idea + 找到 100 个用户后，再请设计师**。那时候设计预算不是问题。

## 设计稿怎么让 AI 帮你检查

AI 不只能做设计，**还能检查你做的设计**。

把 mockup 截图丢给 GPT-4V 或 Claude，prompt：

```
我做了这个 app 的 mockup（[描述产品 + 目标用户]）。请帮我检查：

1. **可读性** — 文字大小、对比度、行间距，用户在 6 寸手机屏上能不能看清
2. **信息层级** — 用户第一眼看到的是什么？主操作按钮够不够明显？
3. **一致性** — 5 个 screen 的设计语言（按钮样式、颜色、间距）是不是统一
4. **文化适配** — 文字、图标、emoji 在中文用户里有没有歧义

给我 5 条具体可改的反馈，每条说明"为什么 + 怎么改"。
```

我自己每次 AI 出设计稿都跑一遍这个检查。**AI 找的问题里大概 30% 我自己没想到**。

## 跟 copy 的关系

这一章讲的是视觉设计，但 **"产品长什么样"和"产品说什么话"是一体的**。下一章专门讲用 AI 写文案 — landing page 标题、产品介绍、邮件序列、客服话术。

讲设计不讲 copy 是因为 — **设计的失败可以靠 copy 救**。比如 hero 图很丑，但标题"Stop wasting 5 hours/week"够狠，用户还是点。

反过来 copy 烂了，再好的设计也救不回来。
