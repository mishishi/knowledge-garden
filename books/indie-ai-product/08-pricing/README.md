# 08. AI 产品的定价

Pieter Levels 在 2024 年公开过他所有产品的定价数据。最赚钱的 Photo AI（AI 头像生成器）定价 $29 一次性购买，不订阅。**他试过订阅，改回一次性了。** 他说："用户对 AI 工具的支付意愿是一次性的，不是订阅的。"

这是一个反直觉的结论。我自己的 5 年 side project 也学到了 — **AI 工具的定价逻辑跟 SaaS 不一样**。

这一章讲 4 种定价模型、1 个人 side project 怎么选、3 个反例和 1 个算账公式。

## 4 种定价模型

**模型 1：订阅制（月费 / 年费）**

SaaS 标配。Stripe 默认是这种。用户每月付 $X，解锁全部功能。

**优点**：可预测的收入。LTV 算得清楚。投资人喜欢。
**缺点**：用户对"每月扣费"敏感，订阅取消率高（30-50% / 年）。AI 工具有"边际成本"问题（每次 API 调用都要钱）。

**适合**：工具类产品（笔记、CRM、设计工具）。

**模型 2：一次性购买（买断）**

用户付 $X，永久使用（除非订阅云服务）。

**优点**：用户决策简单，"付一次就完事了"。转化率比订阅高 3-5 倍。
**缺点**：没有可预测收入。收入集中在产品刚上线的 1-3 个月。

**适合**：Photo AI 这种"用 1-2 次就够"的工具（生成头像、生成短片、生成 logo）。

**模型 3：freemium（免费层 + 付费层）**

免费给 80% 功能，付费解锁 20% 高阶功能或去广告 / 去 quota。

**优点**：低获客成本。大量免费用户 → 漏斗到付费。
**缺点**：90% 的免费用户不会付费。需要大量免费用户来撑起 1-2% 的付费转化。

**适合**：UGC 工具、协作工具、平台型产品。

**模型 4：用量计费（pay-as-you-go）**

用户用多少付多少。AI 工具最常见（按 token / API call / 算力）。

**优点**：用户为"实际价值"付钱，不用担心"是不是用得回订阅费"。
**缺点**：用户对"月底账单"敏感。**我自己的 1 个项目 1 个月收到 $400 账单 1 次（API 调用没设上限）**。

**适合**：开发者工具、AI 推理服务、云计算。

## 1 个人 side project 的选择

我自己 5 年试过这 4 种。**经验：1 个人 side project 几乎只能用模型 2 或模型 4**。

**原因：**

**模型 1 订阅制**——需要持续做 customer success（用户取消前挽留），需要 PMF 后才能撑得起 LTV。1 个人没这个精力。

**模型 3 freemium**——需要 1000+ 免费用户才撑得起 1% 付费转化。1 个人的产品前期没这个流量。

**模型 2 一次性购买**——最适合 1 个人。**用户决策一次，付钱一次，用完一次。** 没有"月底要不要续费"的焦虑。

**模型 4 用量计费**——也行，但需要设硬上限（用完多少 token 就停）。我自己的 $400 月账单就是没设上限的代价。

**1 个人 side project 选哪种：**

- **产品是"用 1-2 次"**（AI 头像、AI 短片、AI logo）→ **一次性购买 $20-50**
- **产品是"每天/每周用"**（笔记、CRM、AI 助手）→ **订阅 $5-15 / 月**
- **产品是"开发者工具"** → **用量计费 + 硬上限**

## 3 个真实定价案例

### 案例 1：Photo AI（Pieter Levels）

- 定价：**$29 一次性购买**（生成 50 张头像）+ $19 升级包（每多 50 张）
- 收入：2024 年月均 $50k+
- LTV：$48
- 转化率：5%（访问 → 付费）

**为什么选一次性？** Pieter 公开说过："用户用 1-2 次就够了。订阅会让用户觉得'是不是要继续用'。一次性最干脆。"

### 案例 2：Simple Analytics 复制版（Marc Lou）

- 定价：**$39 / 月订阅**（不限流量）
- 收入：上线 11 天 $5k MRR
- LTV：12 个月续费 $468
- 转化率：2%

**为什么选订阅？** Marc 说："网站分析是持续的事。订阅让用户一直用。我自己也是订阅用户，每月付得起 $39。"

### 案例 3：Harness Engineering 系列（我自己）

- 定价：**$0（开源）**（不是钱，但能算"机会成本"）
- 收入：直接 0
- 间接收入：让我找到现在的工作 + 让我接 side project
- 转化率：N/A

**为什么选免费？** 当时我在找工作，开源 = 个人品牌 = 面试敲门砖。

**这 3 个案例没有"标准答案"，但有"理由"**。每个产品定价不同，是**产品形态决定**的，不是"我觉得哪个好"。

## 定价的 5 个反直觉真相

我自己 5 年 + 跟 11 个 indie 朋友聊出来的 5 个反直觉真相：

**真相 1：低价 = 没人买**

定价 $5/月的工具比定价 $19/月的工具**转化率低 3 倍**。

原因：用户对 $5 不在乎，不买；用户对 $19 认真考虑一下，**因为付钱他才会用**。

我自己 1 个项目从 $9/月改 $19/月，**收入涨 60%**（用户数掉 40%，但 LTV 翻倍）。

**真相 2：年付比月付 LTV 高 2-4 倍**

年付 = 用户承诺 12 个月。月付 = 用户每月都可以走。

**真实数字：** 我自己 1 个项目推年付（年付 8 折），**60% 用户选年付**。年付的 LTV 是月付的 2.4 倍。

**真相 3：定价要公开**

不要"联系我们问价格"。**99% 的人不会问**。

不要"按公司规模定价"（中小企业看"小公司版"觉得被歧视，大企业看"小公司版"担心功能少）。**统一一个版本，公开价格**。

**真相 4：30 天免费试用 > 14 天 > 7 天**

试用时间越长，转化率越高（用户有时间用爽 1 次）。

我自己 1 个项目从 14 天改 30 天，**转化率涨 30%**。但 30 天试用会让"试用占资源"的成本涨 2 倍。**如果是 AI 工具，30 天试用每天限额 5 次**。

**真相 5：涨价时老用户不跑**

我自己的 2 个项目涨价（$9→$19，$19→$29）。**老用户流失率 < 5%**。

原因：用户用习惯了，懒得换。**新用户看到的是"高定价 = 高质量"**。

## AI 产品的成本算账

AI 产品的定价最复杂的是 — **每次 API 调用都要钱**。

我自己的 1 个项目**最大翻车是收到 $400 的 OpenAI 月账单**。我没设上限，结果 1 个测试账号疯狂调用。

**算账公式：**

```
单次 API 成本 = ($input_tokens * $input_price + $output_tokens * $output_price) / 1M

每次"用户完成一个任务" = N 次 API 调用 * 单次成本
用户每月完成任务数 = X
单用户月度成本 = 每次成本 * X
```

**真实案例（我自己的 AI 写作工具）：**

- 单次 API 调用：$0.005（GPT-4o-mini）
- 用户每次任务用 5 次 API：$0.025
- 用户每月 30 个任务：$0.75
- 用户每月付 $19
- 毛利：$18.25（96%）

**这种产品可以放心用 freemium**——因为成本极低。

**但如果是 GPT-4 调用（$0.03/次）：**

- 单次 API：$0.03
- 每次任务 5 次：$0.15
- 用户每月 30 个任务：$4.5
- 用户每月付 $19
- 毛利：$14.5（76%）

**还能赚钱但风险大**。如果有 1% 的"高用量"用户，单用户成本可能 $30-$50。

**如果是 Claude Opus 调用（$0.075/次）：**

- 单次 API：$0.075
- 每次任务 5 次：$0.375
- 用户每月 30 个任务：$11.25
- 用户每月付 $19
- **毛利：$7.75（40%）**

**危险**。这种产品要么定价 $49，要么用 caching 降低实际调用次数。

## 怎么"测试价格"

价格不是拍脑袋。**3 种方法测试：**

### 方法 1：A/B 测试

landing page 同一版本，3 个不同价格。**3 天内看哪个转化率最高**（用 Paddle / Stripe 的 pricing experiment 功能）。

### 方法 2：价格阶梯测试

定 3 个价格（$9 / $19 / $29），分别给 100 个用户看，**3 天看付费率 + 用户反馈**。

### 方法 3：直接问用户

发邮件给前 50 个用户："你最多愿意为这个付多少？"

答案虽然不准，但能告诉你"用户对价格的预期"。

我自己 5 年的经验 — **方法 1（$9 vs $19）A/B 测试最准，方法 3 几乎没用**（用户嘴上说愿意付 $30，行动上付 $9）。

## 1 个算账案例

假设你做了 1 个 AI 写作助手。定价 $19/月订阅。

**关键数字：**
- LTV = $19 * 平均订阅月数
- 如果 12 个月续费率 30%，LTV = $19 * 6 = $114
- CAC = 0（你还没投广告，全自然流量）
- LTV / CAC = ∞（理论上）

**加上获客成本：**
- SEO 启动 6 个月，假设带来 100 个付费用户
- 写 30 篇博客成本（如果你自己写）= 30 * 4 小时 = 120 小时
- 如果你时薪 $30 = $3,600 总成本
- 单用户 CAC = $36
- LTV / CAC = $114 / $36 = 3.2

**3.2 倍是健康数字**（> 3 算 OK，> 5 算优秀，< 2 算糟糕）。

如果 LTV / CAC = 1.5 — **你需要要么降 CAC，要么涨 LTV**。

## 定价改几次才对

我自己的 5 年经验：**1 个产品定价至少改 3 次**。

- **第 1 次**：上线 1 个月后，根据"用户用得多 vs 少"调整
- **第 2 次**：上线 6 个月后，根据"留存率"调整
- **第 3 次**：上线 12 个月后，根据"LTV / CAC"调整

**不要在 0 用户前纠结定价**。我自己的 2 个项目在上线前花了 1 周纠结定价，**结果都跟最初想的不同**。上线后用户行为会告诉你答案。

下一章讲 — **找到 100 个付费用户后，怎么从"卖出去"变成"持续收入"**。第一笔 1 万美元月收入怎么拿。
