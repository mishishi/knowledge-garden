# 02. 定位：找到你的内容窄门

2024 年 3 月我开了自己的 X 账号，第一篇推文是讲怎么用 Cursor 调 LangChain 的 RAG 流程。配了一张精心画的架构图，用了 40 分钟排版，发出去——14 个赞，3 个评论，其中一个是我自己小号点的。

我以为是我写得不够好。换了个角度又写了一篇：万字长文讲 RAG 评测框架，从 RAGAS 讲到 TruLens，引用了 11 篇论文。发出去——38 个赞，一周后阅读量 800。我盯着后台想，这不对啊，论文我都读完了，图表我自己画的，怎么数据这么平？

那天晚上我翻了一遍竞品。推特上写 RAG 的人少说 200 个，其中粉丝过万的大概 30 个，最头部的那个 @hwchase17 一条推文阅读量稳定 5 万以上。我把他的内容拉出来一条条看：他不讲 RAG 评测，不讲架构图，他讲的是怎么用 LangChain 落地一个真实业务，比如"我给客户做的一个客服机器人跑了 6 个月，召回率从 0.6 干到 0.85，中间踩了什么坑"。

我突然意识到问题在哪了。

我写的不是"用户想看的东西"，我写的是"我会的东西"。这两件事在工程师身上经常被混淆，因为我们受过的训练就是——把一件事搞懂，写文档，输出技术细节。但在内容平台上，用户不欠你一篇论文，他们只关心"这玩意儿对我有没有用"。

这个认知差，我后来用了整整两个月才真正消化掉。这一章想讲的就是我那两个月踩出来的方法：怎么从一个"会做的工程师"变成一个"会被找到的创作者"，核心动作不是写得好，是选得准。

## 我最初是怎么定位的，以及为什么全错了

我最早给自己的定位是"AI 应用工程师，分享实战经验"。这句话看起来没毛病，但放到内容市场里它就约等于"一个普通人"。

2024 年上半年我在 X 上关注了一堆做 AI 内容的账号，慢慢看出一个规律：粉丝涨得快的那些人，都不是"AI 工程师"这个笼统标签，而是某个非常具体的标签。比如 @jxmnop 只讲 LLM 推理优化，@drjimfan 只讲具身智能，@cwolferesearch 只讲 agent eval。他们甚至会在 bio 里写明"我不讲 X，只讲 Y"。

我当时觉得这种窄定位是一种营销策略，写久了自然会宽起来。但实际上 2024 年到 2025 年这波平台算法变了——X 的 For You 推荐逻辑明显更吃"垂直度"，一个账号如果前 50 条推文都聚焦在某个具体子领域，系统的标签会越来越准；但如果今天讲 RAG、明天讲机器人、后天讲股票，账号画像就被打散，推荐池越来越糊。

我后来用 X 的 API 抓过自己和其他几个对比账号的传播数据看。同期发布的推文，垂直度高的账号（80% 内容围绕一个主题）平均曝光是宽泛账号（主题分散 5 个以上）的 3.7 倍。X 的算法不是看一两条爆款，是看"账号整体是不是在持续讲一件事"，这事我到 2024 年底才真正确认。

说白了，平台对账号是有"信用评分"的，垂直度是其中一个核心指标。

但"垂直"和"窄到没流量"之间就隔一层纸。我当时还看过反例：有个账号只讲"MoE 模型的 expert routing 机制"，内容极其硬核，6 个月发了 80 条，粉丝停在 1200。技术深度拉满，但这个话题的潜在读者可能全球就 5000 人，怎么做都做不大。

所以定位这事不是越窄越好，是要找到一个**有付费意愿、有搜索需求、但供给还不太拥挤**的窄门。

## 工程师做定位的三个真实约束

我后来给做副业的朋友讲定位，会先问三个问题，这三个问题不是拍脑袋想出来的，是我自己一个个踩过坑才意识到要问的。

第一个约束是**时间预算**。做副业的人跟全职内容创作者最大的区别是——你每天只有 1.5 到 2 小时。这 1.5 小时还要分配给选题、写稿、排版、回复评论、跟潜在客户聊天。任何一个环节超支，整个系统就崩。

我当时给自己定的是每周更新 3 条推文 + 1 篇长文（3000 字左右）。这个节奏是我试错试出来的：5 条推文我撑不住，质量会掉；2 条推文又太少，账号画像建立不起来。3 条 + 1 篇长文是 2024 年下半年我能稳定维持的极限。

这个时间预算反过来会卡死你的定位。如果你选了一个需要持续做实验才能产出内容的领域（比如 RLHF 训练细节），每周 2 小时根本跑不完一个完整实验，写出来的东西只能是二手转述，质量上不去。如果你选了一个"读一篇论文就能写一篇文章"的领域，产能就拉得起来。

我当时看了下自己 2024 年的输出曲线，发现一个明显规律：我写得最快的几篇，几乎都是"我刚踩过这个坑"的主题；写得最慢的，都是"我研究了一个我之前没接触过的领域"。前者因为有真实经验支撑，后者因为要从零建构认知。选定位的时候，**优先选你已经有 1-2 年真实积累的方向**，不要为了流量去追完全陌生的热点。

第二个约束是**隐私边界**。这是工程师做副业经常忽略的一点。工作中接触的项目、客户、代码、内部数据，这些不是你想写就能写的。我当时在公司做的是企业级 AI 应用，客户名单、架构细节、踩坑记录全在 NDA 底下。刚开始我尝试过脱敏后写"如何给零售客户做推荐系统"，写到第三段就发现，怎么改都有暴露风险，最后整篇废弃。

这逼着我把内容方向转到**我自己的 side project 和公开技术**。比如 LangChain / LlamaIndex 的源码阅读、Cursor / Claude Code 的使用技巧、公开论文的复现。这些东西不属于公司，踩的坑都是自己的，反而写起来更踏实。

如果你是在职工程师想做内容，强烈建议**先把工作内容从你的内容池里彻底划掉**。不是"我小心脱敏一下就行"，是物理隔离。否则你每次想写点什么都要过一遍合规审查，写两篇你就累了。

第三个约束是**变现路径**。做内容最终是为了变现，纯流量号在 2024 年以后的天花板很低。我当时调研过几种变现方式：广告分成（X 的 revenue share 国内几乎拿不到）、付费社群、咨询、课程、引流到自家产品。

不同定位适合不同的变现方式。比如你定位"AI 产品经理"，那咨询和社群合适；定位"AI 编程工具评测"，那引流到 affiliate 或自家产品合适；定位"AI 论文精读"，那付费课程或学术社群合适。

我当时最纠结的就是这个。看了几圈觉得咨询天花板低（一小时 500 到 1500，还要跟客户斗智斗勇），社群需要持续运营、停不下来，课程需要一次性大投入（录课、剪辑、做营销页）。最后我选的是**引流到自家小产品**——一个 Cursor 提示词的 marketplace，月费 9 美元。这个路径前期慢，但一旦内容稳定之后是真正的复利。

这个选择反过来也影响了我的定位：内容方向必须和我的产品有协同。比如我产品是 Cursor 提示词，那内容就不能讲 LangChain（虽然我也懂），必须讲"AI 编程工具链"这个交集——Cursor、Claude Code、Copilot、Continue.dev 这些。**你的内容方向最好和未来的变现产品在同一片土壤上**，否则流量来了也接不住。

## 怎么用 OKR 反向校准定位

OKR 这套东西本来是用在公司管理上的，我搬过来做内容定位发现意外好用。核心想法是：不要先定一个宽泛的"我要做 AI 编程内容"，而是**先定一个 6 个月的可验证目标，再倒推需要什么定位**。

我当时的 OKR 是这样的：

- O（目标）：6 个月内通过内容副业赚到第一个 1 万人民币
- KR1（关键结果）：X 账号粉丝达到 3000
- KR2：发布不少于 24 篇长文
- KR3：至少 30 条推文单条阅读过 1000
- KR4：引流到自家产品的注册用户达到 200

这个 OKR 一写出来，定位的范围就自动收窄了。

先看 KR1：3000 粉丝，6 个月。意味着每个月净增 500 粉。这个量级在"AI 编程"这个大类里不算难，但如果是"AI 编程"下面再细分到"AI 辅助阅读论文"，那可能全球潜在读者都不到 5000，6 个月 3000 粉就太激进了。所以定位必须**在某个相对大池子的边缘，但不能是池子正中央**。

再看 KR3：30 条推文单条阅读过 1000。1000 阅读在 2024 年下半年的 X 上算是中等偏上。要拿到这个数，要么蹭热点（在 Vibe Coding 火的时候发相关推文），要么持续在某个小圈子有话语权。蹭热点不稳定，我选了后者——**找一个相对小但活跃的圈子，深度参与，成为那个圈子里的"常驻嘉宾"**。

我选的是"AI 辅助编程的提示工程"这个小圈子。当时这个圈子在 X 上大概有 50-80 个活跃创作者，读者群体是产品经理、独立开发者、对 AI 工具好奇的传统程序员。这个圈子的特点是：技术深度不算高（不需要懂底层原理），但需要真实使用经验（讲"我今天用 Cursor 做了个 X，踩了 Y 坑"特别有说服力），而且**变现路径清晰**——这部分用户正是我 Cursor 提示词产品的潜在买家。

最后看 KR4：200 注册用户。我产品是 Cursor 提示词 marketplace，那内容必须能持续给这个产品输送精准流量。如果内容方向是"AI 论文精读"，流量来了也对产品没兴趣；如果内容方向是"AI 编程工具链评测"，流量和产品高度匹配。

OKR 这个工具对我最大的帮助不是定目标，是**逼我把模糊的"我想做内容"翻译成可证伪的数字**。一旦数字明确，定位的范围就被圈出来了。

如果你刚开始做，我建议你也写一份自己的 OKR，不用多复杂，1 个 O + 3-4 个 KR 就够。重点是 O 里要有具体的收入或产品目标，不能是"建立个人品牌"这种虚的。

## niche 三角：受众、变现、产能的交叉点

我后来把自己用的定位方法叫 niche 三角。三个角分别是**受众、变现、产能**，你要找的窄门是这三个角都有交集的地方。

受众这个角很好理解：你的内容谁会看？他们的痛点是什么？搜索习惯是什么？我当时用了三个工具来研究受众。

第一个是 X 的高级搜索。输入关键词比如 "Cursor prompt"，按"Latest"排序，能看到最近一两周大家在聊什么、抱怨什么、推荐什么。我当时在搜索结果里发现一个高频抱怨："用 Cursor 写大型项目上下文会乱"，这个问题在我自己的使用中也遇到过。这就成了一个潜在选题。

第二个是 Reddit。我爬了 r/ChatGPTCoding 和 r/cursor 两个 subreddit 2024 年全年的热门帖子，做了个简单的关键词频次统计。"context window"、"agent mode"、"refactor"、"debug" 这几个词的出现频率最高。频次高意味着这个话题的搜索需求大，反过来也意味着供给也多——所以要做差异化，必须在某个高频词下面再切一刀。

第三个是 Amazon 书评和 Udemy 课程评价。这两个地方能反映**付费意愿**。如果某个话题在 Udemy 上有大量"内容浅、踩坑少"的差评，那这是个机会——说明用户愿意为更深入的内容付费。我在调研"AI 编程"时发现 Udemy 上相关课程评分普遍在 3.5-4.0 之间，差评清一色是"内容太基础，都是官方文档能查到的"。这就是个清晰的信号。

变现这个角，关键不是"能不能变现"，是"变现路径和内容调性是否一致"。卖课的内容和卖 affiliate 的内容调性差别很大——卖课需要"权威感 + 完整体系"，affiliate 需要"真实测评 + 场景演示"。两种内容写起来手法完全不同。

我选了 affiliate + 自家产品的混合路径，所以我的内容调性就是"工具实测 + 踩坑记录 + 真实数据"。这种调性对"AI 编程工具链"这个话题特别合适，但对"AI 论文解读"就完全不合适——后者更适合"学术权威 + 通俗翻译"的调性。

产能这个角最容易被低估。很多人都觉得"我每周写一篇就行"，但你真开始写就会发现，不同话题的产能差异巨大。

我后来给自己建了一个产能表，统计了 2024 年下半年每一篇长文的"从选题到发布"耗时。AI 编程工具实测平均 6 小时，AI 论文精读平均 14 小时，AI 产品分析平均 8 小时，AI 行业评论平均 4 小时。

这个数据让我立刻砍掉了"AI 论文精读"这个方向——我每周只有 10-12 小时的可投入时间，如果一篇要 14 小时，那周更就破了。**产能是个硬约束**，不是靠意志力能解决的。

三个角画出来之后，交集其实很小。AI 编程工具链是我当时唯一三个角都能踩到的方向：

- 受众：产品经理、独立开发者、AI 工具爱好者，活跃在 X 和 Reddit
- 变现：affiliate + 自家提示词产品，调性匹配
- 产能：每篇 6 小时，每周一篇可持续

到这一步，定位才算真正定下来。我当时给自己的 bio 写的是："AI 编程工具实测 / Cursor & Claude Code 提示词 / 周更一个踩坑笔记"。三个短句，每个都精准，每个都划界。

## 2024-2026 算法和工具的变化，怎么影响定位

这一节想单独讲讲我观察到的几个重要变化，因为定位这事不是一次定完就完，它要跟着平台和工具的演进而调整。

第一个变化是**搜索流量的崛起**。2024 年下半年开始，X 的 For You 里搜索来源的占比明显提升，到 2025 年据一些第三方数据机构的估算已经占到 30% 左右。这意味着"标题党 + 蹭热点"的作用在下降，"真正能搜到、能解决问题"的内容权重在上升。

这个变化对定位的影响是：**有清晰搜索意图的话题，比纯情绪化话题更值得做**。比如"Cursor 怎么写大型项目"比"AI 让我失业了吗"更有长尾价值。前者一年后还有人搜，后者热度一周就过。

第二个变化是**AI 内容的去同质化压力**。2024 年 GPT-4 刚火的时候，一大堆账号用 AI 批量生成内容、靠数量取胜。到 2024 年底平台明显开始打压，X 引入了一些"原创性信号"算法，Reddit 也开始对纯 AI 内容的子版块做限制。

我后来做过一个实验：同一篇选题，我用纯 AI 生成 vs 自己写再让 AI 润色，两条推文发出去，纯 AI 版的曝光低了 40% 左右（样本很小，只有 20 对，谨慎参考）。但趋势是明确的——**纯 AI 内容的边际收益在快速下降**，有真实经验和个人观点的内容反而越来越值钱。

这反过来影响了定位：如果你选了一个"AI 完全可以替代"的话题（比如基础概念科普、新闻聚合），你的内容很快会被稀释。**选那种"必须有真实使用经验"的话题**，比如工具踩坑、实战案例、数据对比，AI 替代不了你的真实使用感。

第三个变化是**视频和长图的权重调整**。X 在 2024 年底对长图（多图轮播）的推荐权重做了提升，到 2025 年中短视频（30 秒到 2 分钟）的权重也在涨。但纯文字依然有稳定的基本盘。

我当时测试过几种格式的传播效率。同样的选题，纯文字 vs 单图 vs 4 图轮播 vs 短视频，4 图轮播的传播效率最高，单条曝光平均是纯文字的 2.3 倍。这让我后来几乎每条推文都配 3-5 张图。但要注意，**格式是为内容服务的，不要为了流量硬塞图片**。

第四个变化是**AI 工具链的成熟**。2024 年到 2026 年这波最大的红利其实是工具。我自己搭了一套内容生产流水线，包括：

- 选题：用 Claude + 自建的 Reddit 爬虫，每周自动跑一遍高频话题
- 草稿：用 Cursor 或 Claude Code 起初稿，自己再改
- 配图：用 Figma + 一些自动化的图表插件
- 排版：用 Typora 写 Markdown，自动转成适配 X 的格式
- 发布：用 Buffer 定时发布，自动追踪每条推文的数据
- 复盘：每周跑一次 Python 脚本，统计所有推文的曝光、互动、引流转化

这套工具链让我的产能提升了至少 2 倍。如果没有 Cursor 和 Claude Code，我每周写一篇长文就已经是极限了，根本撑不住周更 + 3 条推文的节奏。

工程师做内容最大的优势就是**工具链意识**。普通博主会卡在"我今天没灵感写不出来"，但工程师会想"我能搭个什么工具让这个流程自动化"。这个优势要充分利用。

## 反例：哪些定位走不通

讲完我走通的路，再讲几个我观察到的反例，这些反例比正面经验更有参考价值。

反例一是**"AI 行业评论员"**。2024 年这种账号特别多，每天点评 AI 圈大事，给出"深度分析"。但大部分这种账号 6 个月后就停更了，因为**行业评论的产能要求极高**（每天都要追热点），但变现路径极弱（评论员很难直接卖东西）。除非你本来就在 AI 媒体或 VC 圈有资源，否则这个方向是个坑。

反例二是**"AI 全栈工程师"**。就是什么都讲，从底层 GPU 到上层 prompt engineering。这种账号我关注过几个，粉丝涨得都慢，因为用户搞不清你到底是干嘛的，关注你也不知道下次会看到什么。**内容平台的算法需要清晰的账号画像，全栈等于无画像**。

反例三是**"AI 教学号"**。从 0 开始教小白学 AI，课程大纲式的内容。我当时也试过这种，写了三篇"AI 入门第一课"之类的，反响极差。原因是小白用户的学习路径已经被大平台（极客时间、慕课、B 站头部 up 主）占据了，**新号做教学的冷启动成本极高**，而且教学内容的转化路径长（要建立信任很久才能变现）。

反例四是**"AI 创业日记"**。讲自己怎么创业、怎么融资、怎么从 0 到 1。这种内容看着很香，但有个致命问题——**大部分看的人只是想看故事，不是想用你的产品**。我关注过一个做 AI 创业日记的账号，粉丝 5 万但产品销售惨淡，就是因为粉丝结构不对，都是来看故事的，不是来买东西的。

反例五是**"AI 工具聚合站"**。每天发"今日 AI 工具推荐"，堆砌 10 个工具，每个 100 字介绍。这种账号短期数据看着好看（蹭热点 + 数量多），但**用户粘性极低**，粉丝基本上是来"扫一眼就走"的。算法后期会识别这种低粘性账号，推荐权重会持续下降。

这五个反例的共同点是：**看起来有流量，但实际上变现路径不通，或者产能不可持续**。定位的时候不能只看"这个话题有没有人讨论"，要看"这个话题的讨论者是不是我的目标用户"。

## 今晚能动手的两个事

这一章讲了不少方法论，但方法论不值钱，行动才值钱。最后给你两个今晚就能动手的事。

第一件事：**用 OKR 反向校准你的定位**。拿出一张纸，写下你 6 个月后想达到的具体数字。比如"赚到 5000 块"、"涨到 1000 粉"、"引流到产品 50 个用户"。然后倒推：达到这个数字需要什么内容？要写给谁看？这些人在哪活跃？变现路径是什么？把答案写下来，你就有了初步的 niche 三角。

我当时写这个 OKR 用了大概 40 分钟，反复改了三版。第一版太虚（"建立个人品牌"），第二版太激进（6 个月 1 万粉），第三版才落到一个"跳一跳够得着"的位置。**写 OKR 的过程本身就是定位的过程**，不要跳过。

第二件事：**用 X 高级搜索 + Reddit 抓数据，做一份受众调研报告**。挑 3-5 个你感兴趣的话题，分别搜一下近 3 个月的热度、用户痛点、付费意愿、竞争密度。每个话题写半页纸的分析，最后选那个"数据最漂亮"的方向。

我用了 Claude Code 写了个简单的 Reddit 爬虫，把 2024 年相关子版块的所有热门帖子拉下来做词频统计，整个过程大概花了 2 小时。如果你不会写爬虫，直接用 X 搜索 + 手动复制粘贴也行，慢一点但同样有效。

做完这两件事，你的定位就从"我想做内容"变成"我具体要做 X 方向、写给 Y 人、用 Z 路径变现"。这个具体度就是后面所有动作的起点。
