# 01. 个人品牌的本质:为什么 2026 副业必须做个人品牌

2024 年 3 月,我第一次在 X 上发英文推。那时候我刚被裁,手里揣着 4 个月 buffer,打算花半年时间做一个 AI 工具的副业产品。发推的原因很功利:独立开发者圈子里的人说"You build in public, that's how you get your first 100 users"。

我注册了账号,写了 bio,转发了几个大佬的推,然后看着 follower 计数器从 0 慢慢爬到 12。一周后我放弃了,回去闷头写代码。

这个决定让我多走了 8 个月的弯路。

到 2025 年中,我重新开始做内容的时候,换了完全不同的做法:不是"发推希望有人看到",而是"把个人品牌当作一个产品来搭建"。三个月后我的 newsletter 从 0 涨到 4200 订阅,X 累计涨了 1.1 万粉,第一个付费产品(一份 89 美元的 AI 工程师工具包)卖出了 340 份。

回看这 8 个月的差距,核心不是"我变厉害了",而是我搞明白了一件事:**个人品牌不是"宣传自己",是"建立一种被检索得到的方式"**。

这一章我想把这件事讲清楚:为什么 2026 年的工程师副业,做个人品牌不是可选项,是基础设施;它和"接单"或"做产品"区别在哪;以及作为工程师,我们做这件事有什么独特的优势和坑。

## 你以为的"个人品牌"vs 实际上的"个人品牌"

我先纠正一个常见误解。

很多人听到"做个人品牌"这几个字,脑子里浮现的画面是:每天发朋友圈,晒生活,打造人设,搞流量,然后割韭菜。这是自媒体培训班的剧本,也是大部分人对这件事的第一反应。

但工程师做个人品牌,本质上是另一件事。

我打个比方。你在 GitHub 上提交 PR 的时候,会写 commit message,会写 README,会在 issue 里解释你的设计思路。这些动作不是在"宣传自己",而是在降低别人理解你工作的成本。个人品牌做的是同一件事,只不过载体从代码仓库变成了你在公网上的表达。

具体来说,个人品牌解决三个问题:

第一,信任前置。一个陌生人在决定是否买你的产品、订阅你的 newsletter、或者转发你的内容之前,会先看"你是谁"。这个"看"在大部分情况下只有 3-5 秒。你有没有一个清晰的 profile,你最近在写什么,你过去的项目质量如何——这些决定了这个陌生人是继续往下看还是关掉页面。

第二,渠道复用。当你持续输出内容,你会积累一个"分发网络"——不是某个具体平台的粉丝数,而是多个平台之间相互导流的能力。一篇 blog 发出去,它会同时被 Google 索引、被 X 抓取、被 Hacker News 推荐、被 newsletter 二次分发。这跟"我有一个产品,只能去 Reddit 发帖求曝光"是完全不同的状态。

第三,定价权。同样一个工具,贴上一个没人听过的名字,可能只能卖 19 美元;但如果作者在某个垂直领域有 5000 个活跃读者,他可以卖 89、199、甚至 499 美元,而且还有人会买。这不是割韭菜,这是因为信任降低了买家的决策成本。

我第一次意识到第三点的时候,是在 2025 年 9 月。我当时在 Gumroad 上挂了一个 19 美元的小工具,卖了三周只出了 27 份。然后我把它重新包装,挂到我的 newsletter 订阅者专属的渠道里,定价 89 美元,首周就卖了 64 份。同一个产品,同一个 landing page,只换了渠道和定价,转化率差了将近 6 倍。

这就是个人品牌的杠杆。

## 副业 vs 全职:为什么这条路线是给"不辞职"的人设计的

我必须先讲清楚一件事:我讲的"个人品牌"路径,不是给那些打算辞职全职做内容的人设计的。

如果你打算辞职做全职博主,这件事的天花板和风险都完全不同——你需要的不是 1 万粉,是 50 万粉;你的内容节奏不是一周一篇,是日更;你的收入结构不是"产品 + 咨询 + 偶尔接广告",而是纯流量分成 + 品牌广告。这意味着你得把自己当成一家 MCN 来运营,而不是当成一个工程师在业余时间搞副业。

这一系列的目标读者是另一种人:你在公司上班,工资是你的 base,你想在不影响主业的前提下,花 6-12 个月从 0 跑通一个副业,最终拿到一个月 5000-30000 块的额外收入。

这两种路径的差别,不是"全职更激进、副业更保守"那么简单,而是底层逻辑不一样。

全职做内容,本质是"用内容换广告费"。每 1 万播放量对应 10-50 美元的广告收入,所以你需要的是播放量,内容方向往往越泛越好(泛娱乐、泛科技、泛财经)。这条路竞争最激烈,但上限也最高——头部博主一年能赚 500 万到 5000 万人民币。

副业做内容,本质是"用内容换信任,然后把信任变现为产品/服务/咨询"。每 1000 个真实订阅者,可能对应 10-50 个付费用户,客单价 50-500 美元。所以这条路不追求播放量,追求的是"找到 1000 个愿意为你的专业付费的人"。竞争小很多,但天花板也低很多——大部分副业博主的年副业收入在 5 万到 50 万人民币之间。

我走的是第二条路。我的目标不是"涨到 10 万粉然后接广告",而是"积累 3000-5000 个高信任的订阅者,然后卖 2-3 个 89-299 美元的产品"。这个目标 12 个月内完全可以达到,而且不需要我辞职。

讲清楚这个区别之后,下面的所有建议才有意义。如果你的目标是全职做内容,你应该去研究抖音/小红书的爆款逻辑;如果你的目标是副业搞钱,你要研究的是 newsletter 增长、垂直社区运营、付费产品转化——这套打法完全不一样。

## 工程师做个人品牌的三个具体优势

现在讲讲工程师为什么天然适合做这件事。

我观察到周围做起来的人,大部分都有技术背景。而做不起来的人,大部分是"内容人设型"——他们很会写,但没有产品,没有代码,没有可以拿出来验证的东西。

工程师的第一个优势是:**你能造东西**。

很多做内容的人卡在"我写了很多,但没有产品可卖"。这是因为他们的技能树是"表达",不是"创造"。一个工程师完全不需要担心这个问题——你能写一个 CLI 工具、一个 SaaS 雏形、一个开源项目。哪怕只是一个 200 行的 Python 脚本,只要它解决了某个真实问题,就可以变成一个 19-89 美元的产品。

我自己副业的第一笔收入(2024 年 11 月,640 美元)就来自一个 380 行的 Python 脚本——一个把 Notion 数据库自动同步到 GitHub README 的小工具。我把它发到 X 上,顺便写了一段 800 字的"我为什么做这个"的故事。然后 38 个人买了。这件事我总共花了两个周末。

工程师的第二个优势是:**你能用工具提效**。

我现在一个人维护 newsletter、blog、X、LinkedIn、GitHub 五个渠道,每周产出 2-3 篇内容。如果我手动一个个平台发,我每周要花 8-10 小时。但我现在用了一套自动化 pipeline,实际手动操作的时间是 2-3 小时。具体的工具链我会在后面章节展开讲,这里先列一下:

内容用 Cursor + Claude Code 辅助写初稿,我负责改结构和注入个人经验;发布用一个自建的脚本,把 markdown 同时同步到 Ghost、Hugo blog、dev.to;X 的自动发布用 Buffer;newsletter 用 Beehiiv,内置 RSS 同步功能;数据看板用 Plausible + 自建的 Python 脚本抓 X 互动数据,塞进 SQLite 里分析。

这套东西搭起来花了大约两周,但之后每周省下来 5-6 小时,三个月就回本了。这种"用工具把重复劳动省掉"的思路,只有工程师才有——内容人看到这种 pipeline 第一反应是"太复杂了",工程师看到的第一反应是"这不就是个 cron job 吗"。

工程师的第三个优势是:**你懂数据**。

大部分做内容的人,发完一篇文章就看一眼"阅读量",然后凭感觉决定下一篇写什么。这是凭感觉赌博。

工程师不会这样。我每周会看一组数据:这篇文章的完读率(用 newsletter 的 click-through 估算)、它在 X 上 24 小时内的互动率、它带来多少新的 newsletter 订阅、它后续 14 天里在 Google 搜索的展示次数。

这几个数字加在一起,基本能告诉我"哪类内容值得复制,哪类应该砍掉"。举个例子,我发现我写"工具对比 + 数据"的文章,平均 newsletter 订阅转化率是 8.2%;写"个人故事 + 反思"的转化率是 3.1%;写"纯技术教程"的转化率只有 1.4%。

这个数据彻底改变了我选题的方向——我不再追热点写"如何用 GPT-4 做 XXX",而是反复写"我用了 X 个工具,最后留下了哪 3 个,为什么"。后者更个人化、更难被复制,反而转化率更高。

## 反例:三种工程师常走的弯路

讲完优势,讲讲坑。

我周围认识的想做副业的工程师,80% 走过下面至少一条弯路。我自己三条都走过,所以可以讲得很具体。

第一种:**"我先做好产品,内容以后再说"**。

这是最常见的一种。逻辑听起来很合理:产品是 1,内容是 1 后面的 0;先把产品做好,再做营销。但现实是,如果你不提前 3-6 个月开始建立受众,等产品做出来的那天,你还是从 0 开始冷启动。

我 2024 年走的就是这条路。3 月开始做产品,闷头写到 9 月,产品上线,GitHub 上 3 个 star,Twitter 上 0 个 follower。然后我花了 6 周时间拼命发推、发 Reddit、写 HN 帖子,转化率低到让我怀疑人生。

后来我反思:如果我当时从 3 月开始,边做产品边写"我做这个产品遇到的坑",到 9 月产品上线时,大概率已经有 1000-3000 个订阅者了。这 1000-3000 个人,会直接决定产品的生死。

第二种:**"我要在所有平台都开账号"**。

另一个常见误区。2024 年我刚启动内容的时候,一口气注册了 X、LinkedIn、Medium、Substack、Hacker News、Reddit、YouTube、小红书、微信公众号 9 个平台。然后每个平台每周都发,结果每个平台都做不起来。

原因很简单:时间是有限的,每个平台的内容调性不一样,你不可能同时在 9 个平台都做好。

我后来砍到 3 个核心平台:X(英文,主要做行业曝光)、newsletter(Beehiiv,主要做深度内容和订阅转化)、GitHub(技术社区,做开源项目曝光)。这三个平台的内容互相导流,加起来每周 6-8 小时就能维护好。其他平台如果有自然流量过来,我会手动同步一下,但不主动运营。

第三个平台是后来 2025 年初加的 LinkedIn,因为发现 LinkedIn 的工程师群体购买力更高,适合卖高价产品。但前 12 个月,我只用了 2 个平台,也能跑通 0 到 1。

第三种:**"写技术教程,把自己当 Stack Overflow"**。

这个误区更隐蔽一点。很多工程师做内容,本能反应是"写我懂的技术"。于是满屏都是"如何使用 Python 装饰器"、"如何优化 PostgreSQL 查询"——这种内容。

问题在于,这种内容 Google 已经索引了 10 万份类似的。你的版本,大概率是第 100001 份。除非你有非常独特的视角(比如把 PostgreSQL 优化跟 LLM 的成本联系起来),否则这种内容很难在搜索里冒头,更不可能在社交平台上被传播。

我自己走过的弯路是:2024 年下半年我写了十几篇"如何使用某某工具"的纯教程,平均阅读量 200-500,几乎没有订阅转化。后来我改成"我如何用某某工具解决了某某问题 + 我踩的坑",阅读量立刻涨到 1500-4000,订阅转化率也高一个数量级。

这个差别不是"内容质量"的差别,是"内容定位"的差别。我会在第 02 章专门讲怎么找到这个定位。

## 一个真实的"基础设施"视角

我再说一遍,个人品牌不是营销手段,是基础设施。

怎么理解这件事呢?我讲一个具体的对比。

我认识两个工程师,都是 2024 年开始做副业。第一个叫老周,他做了一个 AI 代码审查工具,产品上线后花了 3 个月做 SEO 和 Reddit 营销,总共带来 80 个付费用户,ARR 大约 6000 美元。他现在已经停更了,因为拉新成本太高,赚的钱还不够他投入的时间。

第二个叫小陈,他在做副业之前,先花了 4 个月在 X 和 newsletter 上写"AI 代码审查"这个垂直领域的内容。2024 年 12 月他的 newsletter 有 3200 订阅,X 有 6800 粉。然后他上线了产品,首月就有 180 个付费用户,ARR 跑到了 2.4 万美元。而且之后每个月自然增长 15-20%,因为他每发一篇内容,都会有一部分订阅者转化成产品用户。

这两位的执行力其实差不多,产品技术含量也差不多。唯一区别是:小陈在做产品之前先搭了基础设施,老周没有。

如果你想搞副业,产品当然要做。但产品是"楼",个人品牌是"地基"。地基没打好,楼盖得越高,塌得越惨。

## 2024-2026 算法与工具的真实变化

最后讲讲为什么这件事在 2024-2026 特别值得做,因为外部环境发生了几个具体变化。

第一个变化:**X(原 Twitter)的算法在 2024 年下半年明显偏向"长内容 + 个人故事"**。

如果你在 2022 年发 X,纯短推 + 一图流最容易爆。但 2024 年开始,平台明显在推 1000-2000 字的长推(可以带图、带代码块、带链接)。我自己的数据:同样 1500 字内容,长推形式的互动率是短推形式的 4-5 倍。

这意味着工程师的一个核心优势——能写长篇深度内容——终于被算法奖励了。

第二个变化:**newsletter 平台的成熟**。

2022 年你要做 newsletter,要么自己搭邮件服务器(劝退 99% 的人),要么用 Substack(作者分成机制不友好)。但 2024-2025 年,Beehiiv、Ghost、ConvertKit 都做得很成熟了,免费版就能用,内置分析、自动化、分阶段付费墙。

我个人用 Beehiiv 一年,总成本 0 美元(免费版够用),订阅者从 0 涨到 4200 全靠自然增长 + 内容导流。这件事在 2022 年几乎不可能。

第三个变化:**AI 工具把内容生产效率拉高了 3-5 倍**。

这一点很关键。我 2024 年初写一篇 2000 字的深度文章,纯手写要 5-6 小时。现在我用 Cursor + Claude Code 辅助,初稿阶段 1-1.5 小时,改稿 + 注入个人经验 1-1.5 小时,总共 2.5-3 小时,产出基本不变甚至更好。

这意味着原本"全职做内容"才能维持的产出节奏(每周 3-5 篇),现在副业状态(每周 10-12 小时投入)完全可以达到。

但我要诚实说一句:AI 辅助写出来的初稿,如果不改,直接发,会有很重的 AI 味——那种"赋能/抓手/沉淀"的大词组合,读者一眼能看出来。我自己的流程是:AI 出 70% 的骨架和素材,我手动改关键段落,确保每个论点都有我的真实案例或数字支撑。完全靠 AI 生成的内容,我试过一次,互动率比手写的低 60%。

第四个变化:**垂直社区的复兴**。

2024 年开始,Lenny's Newsletter(产品方向)、Pragmatic Engineer(工程管理方向)、The Pragmatic AI Tools(AI 工程方向)这些垂直 newsletter 都做起来了,动辄几万到十几万订阅。这跟 2018-2020 年那种"什么都写"的综合博主时代完全不同。

垂直社区意味着更精准的转化、更高的客单价、更稳定的订阅增长。我的 newsletter 之所以能 12 个月从 0 做到 4200,核心是我从第一天就锁定了"AI 工具链 + 独立开发者"这个非常窄的细分,而不是泛泛地写"AI 创业"。

## 给工程师的今晚行动建议

讲完这些,我猜你可能在想:"道理我都懂,但具体第一步是什么?"

两个今晚就能动手的 action:

第一个,**去 X(或者你常用的平台)注册一个独立账号,专门用于你的副业方向**。不要用你的工作账号,不要用你的私人账号。独立的 bio、独立的 handle、独立的定位。这件事 15 分钟能搞定。

为什么是"今晚"?因为大部分人卡在"先想清楚再开始",但实际上你的定位会在做的过程中逐步清晰,不可能在动手前想明白。我自己的定位前前后后改了 4 次,每次都是发了几十篇内容之后才意识到"原来我更适合写这个方向"。

第二个,**打开 Cursor 或者你常用的 AI 工具,写一篇 800-1200 字的"我最近在做的副业项目 + 我遇到的一个具体问题"**。可以是英文,可以是中文,取决于你的目标受众。这一篇不需要完美,不需要有结论,不需要"解决"那个问题。

你只需要把"我正在做这件事"这个信号发出去。我自己 2024 年发的那篇关于"做 AI 工具时如何控制 LLM API 成本"的推文,到现在还偶尔有人看到后订阅我的 newsletter——这就是复利。

如果你今晚就做了这两件事,12 个月后你大概率会感谢现在的自己。下一章我会讲怎么找到你的"内容窄门"——也就是在一个看起来已经饱和的市场里,找到只有你能占住的那个细分位置。
