# 04. 腾讯三连：WorkBuddy + Marvis + CodeBuddy

腾讯在 6 个月内发布 3 款产品——**WorkBuddy（3/9）、Marvis（5/20）、CodeBuddy（持续）**。**WorkBuddy 3 月份月访问量国内第一**。这一章把腾讯系三连全拆。

## 腾讯的策略：分层卡位

腾讯是 15 款里**唯一打"分层卡位"**的厂商——3 款产品分别卡 3 个不同用户层：

- **CodeBuddy**：开发者向（B 端技术团队）
- **WorkBuddy**：企业白领向（B 端全员）
- **Marvis**：操作系统层（C 端 + 个人办公）

**为什么这样分**？

- **腾讯有企业生态**（微信支付、腾讯文档、腾讯会议、腾讯网盘、腾讯乐享）—— B 端是主战场
- **腾讯有 C 端入口**（QQ、微信、应用宝）—— 但 AI 入口的 C 端被豆包抢了
- **腾讯有硬件合作**（中兴 AI 云电脑）—— 操作系统层是船票

## WorkBuddy：月访问量第一的"企业统一入口"

**3/9 上线**——WorkBuddy 来自腾讯云 CodeBuddy 团队。3 月份**月访问量国内第一，环比增速第一**——3 个月人均 Token 增长 10 倍。

### 核心数据（官方披露）

- **3 月月访问量**：国内第一
- **环比增速**：国内第一
- **3 个月人均 Token 增长**：10 倍+
- **SkillHub 收录**：7 万+ Skills
- **2 月 SkillHub 下载量**：3000 万+

### 3 大核心能力：专家 + 助理 + 团队

腾讯云副总裁刘毅在 2026 腾讯云 AI 产业应用大会上公布的"三层能力"模型——**这是 WorkBuddy 最值得讲的产品哲学**：

**第一层：专家（岗位经验封装）**

**把某一类岗位的知识、数据、流程、工具、规范、产出要求**，整体封装成可直接调用的能力包。

**为什么要做专家**：

- 过去员工用 AI 要反复解释"我是什么角色、要做什么任务、遵守什么规范"——**沟通成本高**
- 专家 = **预先沉淀高频上下文**，开箱即用

**实际场景**：

- 销售专家、金融专家、法律专家、财务专家、运营专家、投标专家
- 一个头部游戏公司装了**引擎专家 + 策划专家 + 全栈工程师专家 + 内容创作专家团**
- 覆盖游戏策划、Unreal 代码生成、创意脚本→真机视频→营销素材

**4 大基石**：

- **Skills**：教它怎么做
- **MCP**：教它去哪里做
- **数据与知识连接器**：理解领域 + 行业
- **Harness**：运行底座

**第二层：助理（7×24 数字员工）**

**把专家请进企业数字办公室**——常驻云端，**24/7 在线**。

**关键能力**：

- 跨设备访问（员工 PC / 手机 / 平板）
- 业务系统随时调用
- 记忆和进化（记住用户习惯 + 企业规范）
- 高频经验沉淀成 Skill

**第三层：团队（超级团队）**

**把人和数字员工放进同一个项目空间**——共享上下文、协同推进、复用经验、沉淀资产。

**关键创新**：

- 不只是群聊或任务面板
- 共享上下文的协作环境
- 成员 / 助理 / 聊天记录 / 中间产物 / 业务资料 / 任务进展，**都是团队 AI 理解分析的基础**
- 技能沉淀为**团队 AI 资产**

**核心飞轮**：任务启动 → 团队协作 → 过程沉淀 → 结果交付 → 资产复用。**每转一圈，组织资产变厚一层**。

### 4 类核心产品

**产品 1：CodeBuddy（开发团队）**

- AI 原生开发能力
- 编码 / 任务拆解 / 代码生成 / 验证 / 交付
- 3 种形态：Plugin / IDE / CLI
- 统一治理（知识 / 权限 / 模型 / 研发效率 / 安全）

**实测**：腾讯内部从 30% AI 代码接受度涨到 90% 工程师覆盖，**多数团队 90% 代码 AI 生成，整体编码时间缩短 40%**。

**产品 2：WorkBuddy（业务团队）**

- 写材料 / 做调研 / 分析数据 / 整理会议纪要
- 调用专家 + 助理
- 推进团队任务
- **定位**：不是单点办公助手，**是企业员工每天用的 AI 工作台**

**产品 3：Managed Agents（自建智能体）**

- 企业自建智能体 + 数字员工
- API 接入内部业务系统
- 融入营销 / 财务 / 人事 / 客服流程
- **支持专有云 + 私有云部署**

**产品 4：Agent Suite 智能体套件**

- 腾讯文档（内容生产）+ 腾讯网盘（统一存储 / Agent 外部记忆体）+ 腾讯乐享（企业知识空间）
- **OneID 一次登录 + Credit 统一度量**

### 垂类智能体

WorkBuddy 企业版还接了一批**已经跑出产业价值的垂类智能体**：

- **设计领域**：Miora（创意智能体）、Ardot（设计协作平台）
- **数据领域**：DatabaseClaw（数据库自治）、DataBuddy（数据分析）
- **营销领域**：营销云 + 云 Mall
- **音视频**：音视频 MPaaS
- **金融风控**：天御风控 Agent（情报驱动反诈，**头部大行实战**）

### 核心卖点

刘毅提的核心问题："**员工用了 AI 效率大增，但组织整体效率并未同步提升**"。

**普华永道调查**：56% CEO 表示 AI 投资"一无所获"。

**腾讯的解法**：

- 摒弃"代码行数 / 任务数"等浅层指标
- **核心指标改为"需求吞吐率"**——需求从产生到上线的流通速度
- "**对于软件企业来说，需求吞吐率代表一个组织的创新速度**"

**统一的货币度量体系**：用系数区分不同模型的价格，平台内多应用 + 多模型统一结算。

**即将上线**：任务消耗预估算功能，**执行任务前预判 Credit 使用量**。

## Marvis / Mavis：操作系统层 AI 助手

**5/20 上线**（腾讯应用宝团队）——**这是 15 款里唯一一款"操作系统层" AI 助手**。

### 名字由来

致敬钢铁侠 Jarvis 但用不了（JARVIS 是漫威注册商标），所以叫 Marvis = Marvel + Jarvis。

### 1 主 + 5 副 Agent 架构

**6 个 Agent 各管一摊**：

| Agent | 职责 | 关键点 |
|-------|------|-------|
| **主 Agent** | 拆解任务、调度分配 | 判断是否含多个子目标 |
| **File Agent** | 文件搜索、内容理解、格式转换 | 建本地向量索引，**搜的是语义** |
| **Computer Agent** | 系统配置、硬件信息 | **直接调 Windows API**（不是模拟键鼠）|
| **App Agent** | 软件启停、界面操作 | 可操作安卓 App |
| **Browser Agent** | 网页交互、数据抓取 | 遇到登录墙会暂停等手动 |
| **Search Agent** | 联网搜索、信息汇总 | 多源交叉验证 + 出处链接 |

**关键创新**：Computer Agent **直接调系统 API，不是模拟键鼠**。这意味着：

- 系统操作精准（不依赖屏幕分辨率、窗口位置）
- 很多竞品靠计算机视觉模拟操作，**换分辨率就挂**——Marvis 没这个问题

### 每日 1000 万 Token 免费

**每天白送 1000 万 Token**——**这个额度是 WorkBuddy 之外腾讯最大的补贴**。

**实测三周消耗**：8000 万 Token（按市面价格算几百块）。

### 三周实测两个让人"头皮发麻"的瞬间

**瞬间 1：120 页合同 20 秒出结果**

> 提取所有违约金相关条款，按甲方责任和乙方责任分列，标注所在页码和涉及金额

3 份文件（PDF / Word / 扫描件图片）共 120+ 页。**20 秒出表格**，5 分钟校数字，5:25 发给领导。

**瞬间 2：出门没带电脑，客户急要合同**

周末火锅店，接到客户电话："**上次签的供货合同第三条第二款违约金的表述再看一下**"。

我打开手机微信 ClawBot，**对着手机说"把桌面上的合同文件发给我"**。

- File Agent 10 秒锁定目标
- 主 Agent 发了过来
- 手机震了三下，文件到了
- 转发给客户，**前后不到 3 分钟**

火锅店老板以为我在打手游。

### 关键限制

**最低配置要求**：

- Windows ≥ 6 核 CPU + ≥ 16GB 内存 + 固态硬盘 + Win10+
- Mac Apple Silicon M1+ + macOS 13+

**4 核 8GB 老笔记本不行**——Agent 之间调度抢资源，**响应慢到怀疑人生**。

**机械硬盘不行**——Marvis 本地模式需要加载端侧模型文件，**打开 50 页 PDF 能等十几秒**。

**多端覆盖**：

- 5/20 上线 Android + Windows + macOS
- **6/23 iOS 版本上线**（App Store）

### 4 个真实坑

**坑 1：本地模式性能落差**

"**端侧大模型是腾讯自研 Qwen 系列**"——具体版本未公开。文档理解不错，**创意写作 / 代码生成偏弱**（端侧模型参数量小，**正常**）。

**坑 2：复杂指令翻车**

> 帮我把桌面上最近一周修改过的 Word 文档按日期重命名

**执行了 3 遍才做对**。第一次只重命名部分，第二次日期格式错，第三次终于对。

**多步骤串联建议拆开说**。

**坑 3：功能边界不透明**

> 帮我把这个 PDF 里第三章的图表数据提取出来（能做）
> 帮我对比这两个 PDF 的差异（**做不了**）

**官方没清晰的能力边界说明**——全靠用户踩坑。

**坑 4：隐私焦虑**

效率模式数据走云端——虽然腾讯说加密 + 不训练，**但心理门槛客观存在**。

本地模式解决技术隐私，但没解决信任问题。**腾讯如果公开第三方安全审计报告，会比一万句承诺有效**。

## CodeBuddy：腾讯云的"开发者入口"

CodeBuddy 不是新产品——**2022 年就落地**（最初只服务研发人员）。

### 关键数据

- 2025 年底覆盖腾讯 90% 工程师
- 多数团队 90% 代码 AI 生成
- 整体编码时间缩短 40%
- 研发人员可同时调度 4 名 AI 助理，**并行推进多模块开发**

### 3 种形态

- **Plugin**：VS Code / JetBrains 插件
- **IDE**：独立 IDE
- **CLI**：终端原生

3 种形态都接入 **Admin 后台统一治理**（知识 / 权限 / 模型 / 研发效率 / 安全）。

### 核心优势

**微信小程序深度集成**——WXML / WXSS 语法生成具有"代际优势"。

**错误诊断 + 风险等级评估**——适合腾讯云技术栈团队。

## WorkBuddy + Marvis + CodeBuddy 的协同

腾讯这 3 款**不是 3 场独立战争**——是**1 个企业 AI 平台**的 3 个入口：

```
开发者 → CodeBuddy（写代码）
员工 → WorkBuddy（日常办公）
个人 → Marvis（操作系统层）
企业 → WorkBuddy 企业版（统一管理）
硬件 → WorkBuddy AI 云电脑（中兴合作）
```

**关键判断**：

- **企业级是腾讯主战场**——B 端客户付费意愿强
- **CodeBuddy + WorkBuddy 协同**——开发者 + 业务人员同一平台
- **Marvis 是个人向船票**——通过中兴硬件合作进入 C 端

## 独立开发者用腾讯系的 3 种姿势

**姿势 1：WorkBuddy 主力做 B 端项目**

- 写材料、做调研、分析数据、生成报告
- 跟企业微信 / 腾讯文档 / 腾讯网盘 深度集成
- **B 端项目首选**

**姿势 2：Marvis 主力做个人办公**

- 文件管理、文档理解、系统操作
- **Windows 用户 / Mac M1+ 用户** 适合
- 每天 1000 万 Token 免费

**姿势 3：CodeBuddy 做微信小程序**

- 微信小程序开发者首选
- WXML / WXSS 生成质量国内第一

## 我自己的腾讯系用法

- **写企业项目材料**：WorkBuddy 主力（**比 QoderWork 文档质量略好**）
- **个人文件管理 / 文档理解**：Marvis 主力（**每天 1000 万 Token 不用白不用**）
- **微信小程序开发**：CodeBuddy 主力（**WXML 生成质量真的好**）

**月成本**：0（WorkBuddy 还在内测免费 + Marvis 1000 万 Token/天 + CodeBuddy 基础免费）。
