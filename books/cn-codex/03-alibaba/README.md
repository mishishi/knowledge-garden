# 03. 阿里四面：QoderWork + Qoder CN + 通义灵码 + 钉钉 AI

阿里 5 个月内连发 + 改名 4 款产品——**QoderWork（1/30）、Qoder CN 改名（5/20）、通义灵码（持续）、钉钉 AI（持续）**。**通义灵码是国内 AI 编程市占率第一（800 万用户）**。这一章把阿里系四面全拆。

## 阿里的策略：四面合围

阿里跟字节不同，**没有把所有赌注押在一款产品上**——而是**用 4 款覆盖 4 个细分场景**：

- **QoderWork**：桌面 Agent 办公（B 端白领）
- **Qoder CN**（原通义灵码）：代码 Agent 编程（B 端开发者）
- **通义灵码**：VS Code / JetBrains 插件（开发者向，存量用户）
- **钉钉 AI**：IM 内 AI（C 端 + 内部协同）

**为什么分 4 个**？

- **通义灵码市占率第一**（800 万）—— 不能动这个存量
- **Qoder CN** 是"升级版"—— 承接有专业需求的用户
- **QoderWork** 是"扩展版"—— 桌面 Agent 战场的船票
- **钉钉 AI** 是"C 端入口"—— 钉钉 7 亿用户的潜在转化

## QoderWork：桌面 Agent 的"实习生"

QoderWork 是阿里**最早上线的桌面 Agent**（1/30）——比 OpenAI Codex 整合 ChatGPT 还早 4 个月。

### 核心定位

从 Qoder 代码 Agent 扩到日常办公场景。**底层模型 Qwen 3.7 Max 限时 15 天免费**（目前已延长）。

### 4 大核心能力

**能力 1：任务模式（一次跑完整链路）**

跟豆包任务模式不同，QoderWork 把"任务"做成**项目级别**——左侧是项目列表（每个项目是一个长任务），点进去是"执行步骤 + 产物 + 调用过的 Skill"。

**实际使用**：

> 任务：写一篇苹果 WWDC 2026 的报道文章，附 Word 文档

QoderWork 自动跑：

1. 分析雷科技写作风格
2. 搜集 WWDC 2026 信息
3. 提出 3 个选题角度
4. **等用户确认方向**（关键决策点停顿）
5. 撰写完整文章
6. 生成 Word 文档

**每个步骤在右侧"任务监控"里可见**——这是 QoderWork 比豆包任务模式更工程化的点。

**能力 2：专家套件 + 技能市场**

**专家套件**面向岗位打包能力（法律 / 产品 / 合同 / 投研 / 财税），**安装完整套装就能用**，不用自己拼工具。

**技能市场**更接近插件机制——深入研究、数据分析、PPT 生成、Notion 信息图都在里面。

**6/16 升级的"意识"功能**（记忆/反思/技能进化）：

- 记忆系统：分层结构，主动整理和淘汰冗余信息
- 反思机制：借鉴人脑选择性遗忘
- 技能进化：高频任务沉淀为可复用 Skill
- 所有记忆以**开放 Markdown 文本存放在本地**（**重要**：不锁死格式）

**能力 3：定时任务**

预设模板：

- 午间充电站
- 每周竞品动态追踪
- 每日下载文件夹清理
- 每日数据报表更新

**关键限制**：定时任务需要电脑保持唤醒。**断网 / 关屏就失效**——这是 QoderWork 的硬伤。

**能力 4：应用快照**

捕获当前应用界面为截图 + 可读文本上下文，**让 QoderWork"看见"用户在用的界面**。

**使用门槛**：开启后需要授予 Computer Use、屏幕录制、辅助功能权限，macOS 上首次授权流程可能需要 5-10 分钟。

### 雷科技深度测评结论

我引用雷科技 7.5-8 分的测评，**3 轮测试给到不同分数**：

**第一轮（写文章）**：

- 3500 字 WWDC 2026 报道
- 包含导语、小标题、观点判断、结尾互动
- **问题**：事实有纰漏（"每年 10 亿美元""1.2 万亿参数 Gemini" 等无来源信息）；风格"cosplay" 痕迹明显
- **评分：7.5**

**第二轮（做 PPT）**：

- 13 页《雷科技商务介绍.pptx》
- 主动发现缺 Node.js 环境，**自己下载安装**
- **问题**：没用真实 logo（用 AI 插画代替）；目录页有"05 I am the chapter name" 模板残留；数据无脚注
- **评分：7.5**

**第三轮（做网页）**：

- IFA 2026 展会专题网页，HTML + CSS + JS
- 7 个 section、响应式、汉堡菜单、0 出错
- 写**真实可运行代码**（不是截图）
- **问题**：仍然用 emoji + 蓝色方块代替真实 logo
- **评分：8**

**总结**："**QoderWork 已经实现了从'只会回答'到'能干活'的阶段性跨越，但目前成品的质量可能需要多抽几次卡，多调整一下才能投入到工作流**"。

**核心定位**："**AI 实习生**"——能干活，但未必干得好。

## Qoder CN：通义灵码的"专业升级"

**5/20 改名**——从"通义灵码"升级为 Qoder CN 系列。

### 3 大子产品

- **Qoder CN IDE**：AI 智能体编码平台（VS Code 风格）
- **Qoder CN JetBrains/VS Code 插件**：编辑器集成
- **Qoder CN CLI**：终端原生形态
- **QoderWork CN**：桌面应用（与 QoderWork 协同）

### 3 大核心能力

**能力 1：上下文工程（10 万文件检索）**

内置代码检索引擎，**一次可检索 10 万个代码文件**——支持跨文件、跨模块定位符号、调用链与依赖。

**Repo Wiki**：将项目中隐性的知识（架构意图、模块职责、设计理念）显性化。**解决"只有 Bob 知道"的问题**。

**能力 2：长短期记忆系统**

保存用户偏好、团队规范、历史决策等上下文，**自我进化**——比如自动记住"生成单元测试"或"生成任务报告"等习惯。

**能力 3：最优模型动态路由**

集成 Claude / GPT / Gemini / 通义等多个模型，**根据任务类型、语言语境和性能预算自动选择最合适的模型**。

**核心差异**：市面很多"多 Agent"是同一个 AI 换 prompt，**Qoder CN 的 Expert 是经过专项调优的 SWE Agent**——不同专家自动路由到不同模型，规划用 Opus、写代码用 GLM5、浏览器测试用 Kimi K2.5。

### 专家团模式（Experts Mode）

**Qoder CN 最新出的杀手级功能**——你给一个需求，AI **自动组建一支专家团队**。

**架构**：

- **Leader Agent**：拆任务、组团队、盯进度
- **专家成员**：前端开发、后端开发、测试工程师、代码审查、技术调研员
- **每个 Expert 是独立 SWE Agent**，互不干扰

**实战演示**（用户跑的小学生英语单词听写项目）：

1. 用户给模糊需求 → Leader 询问 5 个澄清问题
2. Leader 规划实施方案
3. 同时启动后端工程师 + 前端工程师（并行）
4. Leader 把前端再拆 3 个工程师（同时跑）
5. 全部完成后，**测试工程师 + 代码审查员加入**（并行）
6. 评审发现接口不一致，紧急召唤后端工程师修复
7. 测试工程师自动打开浏览器一步步测试
8. 项目完成，**0 Bug**

**关键观察**：

- 不是"单 Agent 假装多 Agent"——是**真正独立上下文的多 Agent 协作**
- Leader Agent 动态调度，**任务拓扑自动管理**
- 适合复杂多步骤任务（不适合简单单文件改动）

**使用门槛**：Pro+ / Ultra / Teams 用户可直接使用，Pro 用户需加入 Waitlist。

## 通义灵码：800 万用户的"版本答案"

**通义灵码 = Qoder CN 改名之前的名字**。改名后通义灵码变成 Qoder CN 系列的一部分，**但老用户还在用通义灵码插件**。

### 核心数据

- **800 万用户** —— 国内 AI 编程市占率第一
- **多文件联合理解**（2026/3 发布）—— 解决 AI 编程最大痛点"上下文断裂"，代码生成采纳率提升约 35%

### 适合场景

- **企业级 Java/Go 后端开发** —— 跟 Spring Boot / Alibaba 中间件深度集成
- **追求生态集成的团队** —— VS Code / JetBrains 全覆盖
- **中小团队快速接入** —— 老用户基础大

**为什么能赢**：

- 先发优势（2023 年就推出，积累用户）
- Java/Go 生态深度优化（懂 Alibaba 中间件）
- 工程记忆（记住项目历史决策）
- 国产化合规（数据不出境）

## 钉钉 AI：7 亿用户的潜在转化

**钉钉 AI 个人版** 不是新东西，**2023 年就推出**——但 2026 年开始发力"AI Agent"能力。

### 核心场景

- **IM 内 AI 助手**：发消息、订日程、查文档不用切应用
- **AI 客服**：企业用 AI 自动回复
- **AI 会议助手**：自动记录 + 总结 + 待办

**关键差异**：钉钉 7 亿企业用户基数。**C 端 AI 入口之争，钉钉是必争之地**。

## 4 款产品的组合策略

阿里这 4 款不是 4 场独立战争——**是 1 场战争的 4 个阵地**：

```
企业开发者 → 通义灵码（存量）→ Qoder CN（升级）
                       ↓
                QoderWork（桌面 Agent 扩场景）
                       ↓
企业员工 / 用户 → 钉钉 AI（C 端 + IM 入口）
```

**关键判断**：

- **开发者向**：通义灵码 → Qoder CN 升级（**留住 + 升级**）
- **白领向**：QoderWork 拉新（**桌面 Agent 战场**）
- **C 端向**：钉钉 AI 转化（**IM 入口**）

**这个矩阵对阿里是优势**——4 款产品覆盖 4 类用户，**相互引流**。

## QoderWork 的 5 个真实坑

**坑 1：事实核验仍需人工**

写文章时给的数据**有 30% 概率不准**（AI 编数据是常见问题）。**必须人工核**。

**坑 2：风格"cosplay" 痕迹**

学某个写手风格会**学到表面**（高频词、口头禅），**学不到内核**（判断、信息密度、节奏）。

**坑 3：模板残留**

PPT 偶尔有"05 I am the chapter name" 这种英文占位符没清理。

**坑 4：定时任务依赖电脑唤醒**

**断网 / 关屏就失效**——这跟"24/7 AI 员工"宣传不符。**跑长任务要电脑一直开着**。

**坑 5：应用快照授权门槛高**

macOS 上首次授权要 5-10 分钟，**不是所有人愿意给**。
