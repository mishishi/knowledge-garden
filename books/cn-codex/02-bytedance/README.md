# 02. 字节三连：TRAE Work + 豆包 + 豆包专业版

字节在 6 个月内连发 3 款产品——**TRAE Work（6/9）、豆包任务模式（6/12）、豆包专业版（6/24）**。这一章把字节系一锅端。

## 字节的策略：双线作战

字节在桌面 Agent 战场**打两条线**：

- **B 端（开发者向）**：TRAE Work — IDE + 桌面 App + 三端协同
- **C 端（用户向）**：豆包任务模式 + 豆包专业版 — 移动端 + 办公场景

**为什么分两条线**？

- TRAE Work 承接 SOLO 的开发者用户
- 豆包承接 C 端用户基数（**国内最大**）
- 中兴 AI 手机合作让豆包进入硬件入口

## TRAE Work：3 模式 + 3 端协同

TRAE Work 是字节系**最值得独立开发者用的产品**——**中国版完全免费，个人版全功能**。

### 3 种工作模式

**模式 1：Work 模式（需求对话）**

启动后给个模糊需求：

> 帮我做一个咖啡品牌官网

TRAE 进入需求引导模式——**主动追问细节**（品牌定位、目标用户、风格偏好、参考竞品），最后输出**PRD 需求文档 + UI 设计规范**。

实测：5-10 分钟出完整 PRD。**比跟产品经理来回拉扯快 10 倍**。

**模式 2：Design 模式（设计 + 出图）**

TRAE Work 在 6 月底新上的功能——**专门做设计**。三条能力：

- **Design Library**：支持 Figma 解析 / 设计规范导入 / 风格探索
- **改图三种方式**：对话调整、鼠标框选、面板调数值
- **导出**：Figma 精修 / Code 模式

实测：把 Google Material Design UI Kit 丢进去，**30 分钟** 扒出主题色板、字体层级、组件样式，生成 Design Library。然后基于这个 Library 出音乐 App 首页，**严格遵守设计系统**（不是开盲盒）。

**模式 3：Code 模式（落地代码）**

一键从 Design 跳到 Code。**Work 聊需求 → Design 出稿 → Code 导代码**，**全程不切平台**。

实测：完整流程（含需求 + 设计 + 落地）**1 小时以内**。传统流程（产品 + 设计 + 前端）3 天起步。

### 3 端协同：手机 + 电脑 + Web

**这是 TRAE Work 跟其他产品的最大差异**。

**痛点**：你必须坐在电脑前才能干活。但碎片时间（通勤 / 等会议 / 排队）每天 30-60 分钟就浪费了。

**解决**：

- **手机 App**：语音下发任务（按麦克风说话，识别率 95%+）
- **电脑端**：实时同步任务进度，可视化 AI 操作过程
- **Web 端**：跨设备访问

**典型场景**：

- 周五下班地铁上，语音下任务："整理竞品动态并生成周报"
- AI 在公司电脑后台跑
- 到家看结果，**碎片时间从"浪费"变"提前开工"**

### Cloud 云端 Agent

**最黑科技的能力**——手机端切换到 Cloud 环境，选 GitHub 仓库，**SOLO 在云端自动拉代码、创建分支、写代码、跑测试、推 PR**。

电脑全程不需要开机。

**实测**：6 月份有开发者靠这个 8 小时跑出 47 个子 Agent 并行（详见 05 章 Kimi Work 对比）。

### Skills 系统

TRAE 桌面端安装的 Skill，**手机端也能调**——但**手机端看不到调用过程**，只能从"思考过程"里看到 Skill 被调用。

**手机端 4 个限制**：

- 不支持 Mermaid 图表渲染
- 代码框显示不友好
- 看不到 Skill 具体位置
- 工作目录浏览受限

**实战建议**：手机端适合**轻量任务**（语音发任务、查进度、approve），重活回桌面端或 Web。

## 豆包任务模式：C 端最大入口

**豆包** 是字节的 C 端 AI 入口。**任务模式**是 6/12 上线的能力——完整执行任务，而不只是问答。

### 核心能力

- **任务执行**：直接操作本地应用 + 联网
- **多步推理**：从"问答"升级到"代办"
- **场景化模板**：预设高频场景（订餐、查天气、做攻略）

### 实际使用

豆包 App 升级到最新版本后，对话框里多一个"任务模式"开关。开启后：

> 帮我把这周公众号所有文章标题整理成表格，按阅读量排序

豆包会：

1. 读取公众号文章列表
2. 抓取每篇阅读量
3. 生成表格
4. 推送到飞书

**优势**：用户基数大（豆包是国内 C 端 AI 第一），任务模板多。

**劣势**：最近受"豆包付费消息"争议影响（被批"AI 强制推消息"），**6/24 豆包专业版发布** 部分为了救场。

## TRAE Work 实战案例：1 小时出完整咖啡品牌官网

我跑了完整流程，把 prompt 写出来给你参考：

### 步骤 1：Work 模式

> 我要做一个咖啡品牌官网，品牌名叫"山与杯"，定位精品咖啡，目标用户 25-40 城市白领。

**TRAE 追问 5 个细节**：

1. 品牌调性？（答：简约 + 温暖）
2. 核心产品？（答：咖啡豆 + 挂耳包）
3. 销售渠道？（答：官网 + 第三方电商）
4. 主要内容板块？（答：品牌故事、产品展示、门店信息）
5. 参考竞品？（答：Manner、% ARABICA、三顿半）

**输出**：PRD 文档 + UI 设计规范。

### 步骤 2：Design 模式

切到 Design 模式，说：

> 按照刚才的 PRD，做一个官网首页。要求：简约风、大量留白、首屏用山脉意象。

**Design 模式出图**：

- Hero 区：留白 + 山脉 + "从山到杯" 主标题
- 产品区：咖啡豆 / 挂耳包两个品类
- 门店信息
- 简洁导航

**框选微调**：

> 把 Hero 主标题字体放大、加粗、字间距加宽
> 背景换成浅米色

**面板微调**：颜色色值、字号、间距直接拖。

### 步骤 3：Code 模式

一键切到 Code：

> 把这个设计稿导成 Next.js + Tailwind CSS 组件

**输出**：

- `app/page.tsx` — Hero 组件
- `components/hero.tsx`
- `components/product-card.tsx`
- `tailwind.config.js`（含品牌色）
- 完整可运行项目

### 总耗时

- Work 模式需求引导：8 分钟
- Design 模式设计稿 + 微调：25 分钟
- Code 模式代码生成：20 分钟
- **总计 53 分钟**

**传统流程**：产品经理 1 天 + 设计师 2 天 + 前端 1 天 = 4 天。

**效率提升 ~100 倍**。

## TRAE Work 的 5 个真实坑

我自己测了 3 周，遇到这些坑：

**坑 1：海外项目适配差**

TRAE 在国内项目（Next.js / 微信小程序 / 钉钉）上**很强**，但遇到海外项目（Stripe / AWS / GCP）就会卡。

**解决**：海外项目用 Codex CLI 主力，TRAE Work 辅助。

**坑 2：大型超复杂项目深度定制能力有限**

TRAE 擅长中小型项目和快速原型，**几百个文件的大型 monorepo** 它会跑偏。

**解决**：大型项目用 Codex CLI + worktree，TRAE Work 只做 UI 改稿 + 单组件生成。

**坑 3：手机端渲染能力有限**

手机端看不到 Mermaid 图表、代码框显示不全。

**解决**：手机端只做"语音发任务 + 查进度"，重活回桌面。

**坑 4：Skill 调用过程不透明**

手机端看不到 Skill 在哪调用，只能看"思考过程"。

**解决**：重要 Skill 任务回桌面端查看完整流程。

**坑 5：偶尔"答非所问"**

中文模糊需求偶有理解偏差（虽然行业领先，但不是 100% 准确）。

**解决**：需求写具体一些 + 给出验收标准 + 让它先出计划再写代码。

**姿势 1：TRAE Work 作为主力（推荐中文项目）**

- 用户基数大
