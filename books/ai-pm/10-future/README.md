# 第十章 未来十二个月 AI PM 怎么变

去年十月, 我带的一个 PM 团队, 十二个人, 做了一个我自己都觉得有点难堪的实验: 让组里三个人用纯 AI 工具链 (Cursor + Claude + 一堆自建 agent) 跑两个迭代, 剩下九个人按老流程 PRD + review + 排期, 最后对比三个维度的数据: 需求交付周期, bug 率, 业务方满意度.

结果不太好意思说. AI 流程那组的两周交付量, 是老流程那组的三倍, 业务方满意度 4.6 vs 3.9, bug 率还低 0.4 个百分点. 那三个人里有俩是入职不到一年的新 PM, 连上线流程都没完整跑过那种.

当时我坐在会议室里看数据, 第一个反应不是"好, 牛逼", 而是"妈的, 我过去八年攒的很多东西要归零了". 不夸张. 这一章就是把我从那个反应开始, 到现在想清楚的一些东西写下来. 包括哪些事我想错了, 哪些路径我自己走过但建议你别走, 以及如果你今天还是一个 PM, 这一周可以动手做点什么.

## 哪些 PM 工作正在被 agent 吃掉

先说结论再说过程, 因为这块大家问得最多. 我自己的判断是, 未来十二个月, 至少有四块传统 PM 工作会被 agent 实质性地吃掉一半以上, 有一块会彻底消失, 还有一块反而会变贵.

第一块, PRD 初稿. 这个其实从 2023 年底就开始了, 但 2025 年彻底成熟. 我现在写 PRD 流程是这样的: 拿到一个需求 brief, 先丢给 Claude 跑一版, 自己改一轮, 再丢回去 review 一轮, 出来的东西比我 2022 年从零写好 30%-40%. 注意我说的是"比我从零写好", 不是"比我写得好". 区别在哪. 比我从零写, 我不用想结构, 不用查遗漏的边界 case, 不用在格式上花时间. 这些"非思考性劳动"是 agent 最擅长的. 真正需要我拍板的还是: 这个需求到底做不做, 边界在哪, 跟其他需求的依赖怎么解. 这些东西 AI 给不了, 至少这十二个月还不行.

第二块, 竞品分析. 以前一个 PM 拿一周时间做一份完整的竞品分析报告, 现在用 agent + 抓取 + 总结, 两小时能出 70% 的内容. 剩下 30% 是判断: 这家公司的策略意图是什么, 这套 UI 是临时方案还是长期方向, 他们最近融资 / 招人 / 发 PR 在暗示什么. 这些判断 AI 做不了, 因为它没有 PR 渠道, 看不懂组织变动的弦外之音. 但 70% 那部分, 老实讲, 大部分 PM 没做得比 AI 好, 只是做得比 AI 慢.

第三块, 数据分析里的"描述性"部分. 上周我看一份 DAU 下降的分析, 团队里一个 PM 用 SQL 拉数据写了 600 字描述, 用了四小时. 我让他用 Claude + 内部数仓的 agent 重做, 十五分钟, 出来 800 字, 多写了 30% 内容, 还附了一张自动生成的图表. 这种描述性 analysis, 过去是 PM 入门的练手活, 现在是 agent 的基本盘. 我跟团队说, 这块以后你做的只能是"提问者", 不能是"作答者".

第四块, 会议纪要 + 跟进事项. 这个最没技术含量, 但吃掉的是 PM 一天里最碎的那两小时. 飞书 + 通义听悟 + 各种会议 agent, 早就能干这活了. 我现在开会基本不带脑子记, 散会看 agent 输出, 自己改 20%, 完事. 这块我建议所有 PM 这周就开干, 不要等.

会被彻底消失的那一块, 是基础的需求池整理. 以前 PM 一项重要工作是从用户反馈 + 业务方需求 + 数据里, 整理出 backlog, 给每条打优先级. 2024 年这块还有价值, 2025 年开始 agent 已经能基于多源输入自动出 backlog, 准确率大概 70% (我自己测的, 跑了大概六百条样本, 用我们组过去一年的真实需求). 这块我的判断是, 未来十二个月, "整理 backlog" 这件事不会再作为一个独立工作存在. 它会变成 agent 输出 + PM 拍板的合并动作.

反而会变贵的那一块, 是定义问题. 说白了, 是"在所有看起来都能做的需求里, 识别出哪 0.1% 是真正值得做的". 这事 AI 干不了, 因为它没有"判断一个事值不值得"的能力, 它只会判断"一个事怎么做得对". 过去这块能力在 PM 群体里是被低估的, 大家觉得是"老板 / 业务方的活". 接下来十二个月, 具备"定义真问题"能力的 PM, 我预估薪资溢价会在 30%-50% (这个数我没有数据支撑, 是看招聘市场 + 跟几个猎头朋友聊出来的, 你当参考). 因为稀缺, 所以贵.

## 哪些 PM 价值在 agent 替不了的地方

我去年跟一个做了十五年 PM 的前辈吃饭, 他说了一句话我记到现在: AI 替代的不是 PM, 是 PM 里那部分"流程操作工"的部分. 剩下来的, 反而是 PM 这个职业最值钱但被低估最久的部分. 我同意. 我自己拆了一下, 大概是四块.

第一块, 跨团队共识. 你把三个部门负责人拉到一间屋子, 让他们就一件事达成一致, 这事 AI 做不了. 我不是说 AI 不能写会议纪要, 是说"让三个利益不一致的人在四十分钟里对齐到能往下走"这件事, 涉及到权力关系, 信任账户, 既往的恩怨, 这些 AI 看不见也学不会. 我在字节那会儿, 有段时间我自嘲是"会议室保姆", 天天帮人拉对齐. 当时觉得这事 low, 现在回头看, 这事才是 PM 这个职业真正的护城河.

第二块, 模糊需求的判断. 业务方跟你说"我要一个 AI 助手", 这句话背后有二十种可能的产品形态. 哪种对, 怎么从这二十种里挑出当下该做的那种, 这事 AI 给不出答案. 它能给的是"每种形态的优劣分析", 但"当下该做哪种"这个判断, 牵涉到组织能力, 当前资源, 老板的耐心, 业务的战略窗口. 这些都是 PM 脑袋里装着但写不到 brief 里的隐性知识. 我自己判断这块价值未来十二个月会涨, 但前提是你得真的在做"判断"而不是在"传话".

第三块, 关键节点的取舍. 上线前一天, 数据突然掉 5%, 业务方在群里 @ 你, 老板在问要不要回滚, 工程师在等你拍板. 这种时刻, AI 帮不了你. 不是技术上帮不了, 是它没有"承担这个决定"的能力, 也没有"为这个决定的后果负责"的身份. 我自己踩过的一个坑: 有一阵子我养成习惯, 遇到关键节点先问 Claude 意见. 后来复盘发现, Claude 永远倾向于"稳妥但保守"的选择, 因为它训练目标就是这样. 但真实的商业场景里, 有一半的正确选择是"不那么稳妥"的那个. 那个判断只有人能做, 而且只有"在这个业务里泡了足够久"的人能做.

第四块, 对人的理解. 我做了十一年 PM, 自认为最值钱的两个能力, 一个是上面说的判断, 一个是这个: 能在五分钟内判断一个人在这个组织里真正想要什么. 业务方跟你拍桌子, 真的是因为这个功能没做? 还是因为他下个季度的 KPI 没着落? 工程师跟你较真一个技术方案, 真的因为方案有问题, 还是因为他觉得自己的位置感被威胁? 这些, 任何 AI 都读不出来. 因为它是组织里的一台机器, 不是组织里的一个人. 我在这件事上栽过跟头, 2021 年带字节一个团队, 我以为一个工程师是真的对方案有意见, 跟他 battle 了两个月, 后来才明白他是对我抢了他 leader 的位置感不满. 那次教训告诉我, PM 的核心技能里, "读人" 至少占三成.

## 几个具体的趋势判断 (可证伪的那种)

我接下来要说几个具体判断, 都是关于未来十二个月, 不是未来五年. 因为五年那种预测我也做不出来, 而且我做的那些五年预测后来都证明是错的. 十二个月的预测, 至少能让我在错的时候复盘出点东西.

第一个判断, agent 工具链会从"散件"收敛到"工作流". 2024 年 PM 用 AI 工具, 是一堆散件: 这个写文档, 那个画流程图, 那个查数据, 那个整理会议纪要. 2025-2026 年, 这些会收敛成几个主流的"PM 工作流产品", 一个产品覆盖你从需求接收到上线的全流程. 我自己已经在用 Notion AI + 飞书智能伙伴 + 几个内部 agent 搭的工作流了, 节省的时间每天大概两到三小时. 但这块还在早期, 产品形态不稳定, 选型要谨慎. 我的建议是不要押注单一产品, 把核心工作流留给自己能控制的工具 (比如本地 Claude + 自己写的脚本), 把协作类的工作流再用 SaaS 产品.

第二个判断, PM 的"技术理解"门槛会从"懂原理"降到"懂使用边界". 以前招 PM, 我会问"你能不能讲清楚 RAG 的基本原理", 现在这个问题反过来: 我会问"你知道 RAG 在什么场景下不能用吗". 未来十二个月, PM 不需要懂 RAG 怎么实现, 但必须知道什么时候不能用 RAG. 这种判断能力的差距, 会比"懂原理"那种差距更值钱, 也更难培养. 我自己带新 PM 的方式也变了, 以前让他们读论文, 现在让他们做失败案例的复盘. 复盘十个失败的 AI 项目, 比读十篇 paper 收获大.

第三个判断, 跨职能 PM (懂业务 + 懂 AI + 懂工程) 会比纯产品 PM 稀缺 3-5 倍. 理由很简单: 业务方懂业务的, 一抓一把. 工程师懂工程的, 也一抓一把. AI 算法懂算法的, 现在也多. 但同时懂三件事, 还能在三者之间做翻译的, 极少. 我自己过去两年招 PM, 优先看的就是这三种能力的交集. 有一个面试者我印象很深, 之前是数据分析师, 自己用 LangChain 搭过客服 agent, 又在我们目标客户的行业里卖过三年 SaaS. 这种人我直接发 offer, 不用聊二面. 但这种人市场上可能不到 5%, 而且越来越贵.

第四个判断, 文档型 PM 的末日. 这个我比较激进, 但我坚持. 未来十二个月, "会写文档" 不再是 PM 的核心能力. "能产出被 agent 正确理解的 brief" 才是. 区别在哪. 写文档是"把你脑子里的东西倒出来", 写 brief 是"把 agent 能用的结构化信息喂进去". 后者需要的是结构化思维, 不是文笔. 我自己现在写 brief, 70% 的时间花在"让信息结构化", 30% 才花在"组织语言". 这个反过来, 我跟新人说, 未来你简历上"文档能力"这个标签, 该换成"信息结构化能力". 不换, 容易被 agent 时代的招聘系统自动筛掉.

第五个判断, PM 的 OKR 会从"交付指标"转向"判断指标". 这个变化是必然的, 但什么时候发生, 我判断在 2026 年中之前会大面积出现. 现在大部分 PM 的 OKR 还在写"完成 XX 个需求", "上线 XX 个功能". 这些都是过程指标, 容易被 agent 替代后变成"agent 完成了 XX 个需求"这种话术. 真正的 PM OKR, 应该回到"我做了 XX 个判断, 这些判断带来了 XX 的业务结果". 这种 OKR 写起来难, 评估起来更难, 但它才是 PM 这个职业在 agent 时代的真正价值锚点.

## 个人路径, 给你这周就能开始做的两件事

这块我不打算写成"十个建议", 那是鸡汤. 我只给两个, 都是我自己这周还在做, 而且我自己验证过有效的.

第一件, 这周开始, 把你的任何一个重复性 PM 工作, 跑一遍 agent 流程. 不要从大件开始 (写 PRD / 做调研), 从小的开始. 比如: 每天的会议纪要, 每周的周报, 每次的 bug 复盘文档, 每次的数据周报. 拿一个, 跑通 agent 流程, 哪怕只是 30% 的部分自动化, 都能让你建立对 agent 能力的真实体感. 我见过太多 PM, 嘴上说"AI 改变了我的工作", 但具体问他怎么用的, 一问三不知. 这种 PM 接下来十二个月会很被动. 你不需要精通, 你需要的是"用过"和"知道它的边界在哪".

我自己就是从周报开始的. 当时是 2024 年初, 每周写周报要花我三小时, 后来我用 Claude + 一个简单的 prompt 跑, 现在每周周报十五分钟搞定, 而且写得比我自己写的好 (它会把"我做了什么"和"带来了什么"分得很清楚, 这是我自己的写作盲区). 这一件事一年下来, 给我省了 130 多个小时. 这个时间我没用来"摸鱼", 我用来多读了六本产品相关的书, 多跟业务方吃了十顿饭, 多想了几个我之前没时间想的产品方向. 边际收益巨大.

第二件, 这周开始, 找三个你佩服的 PM (不一定是你们公司的), 仔细读他们过去一年做的项目复盘, 重点看"他们怎么做的判断"而不是"他们怎么做的事". 我去年花了两周读了二十多个 PM 的复盘文档 (大部分是我在脉脉 / 即刻 / 各种 PM 社区上扒的, 公开的), 最大的收获不是学到什么方法, 是看到这些 PM 在关键节点上"为什么选 A 不选 B". 这种决策日志, 比任何 PRD 模板都值钱. 而且这种东西 AI 给不了, 因为它是个人经验, 是不可迁移的隐性知识, 是"这个 PM 在这个组织这个业务里这一刻的判断". 你读十个, 你的判断水位就会被拉高. 你读三十个, 你会开始形成自己的判断框架. 我自己做产品判断的时候, 脑子里会自动跑过这三十个案例里类似的场景, 然后问自己"那些 PM 当时怎么选的, 我现在能不能选一样". 这种"脑内 benchmark"是 PM 真正的护城河, agent 学不走.

最后说一句不太好听但真心的. 未来十二个月, 焦虑的 PM 会越来越多. 因为工具在变, 组织在变, 业务在变, 老板对 PM 的预期也在变. 但我自己的判断是, 真正的好 PM, 不会被淘汰. 淘汰的是那种"把 PM 当成流程操作工"的 PM. 你如果过去几年一直在做"接收需求 → 写 PRD → 跟进排期 → 验收上线"这种循环, 你确实应该焦虑. 但你如果过去几年一直在做"判断真问题 → 拉齐共识 → 承担关键决定 → 复盘学习", 你不会. 反而, 工具变强之后, 你的杠杆率会指数级上升.

我去年十月那个实验, 那三个用 AI 工具的新 PM, 后来两个被提拔了. 不是因为他们用了 AI, 是因为他们用 AI 节省下来的时间, 全部投入到了"判断"和"读人"这两件事上. 另外一个没用 AI 工具, 但也没把时间花在判断和读人上, 半年后走了. 你看, 工具从来不是决定性因素, 决定性的是你把时间花在哪.

这周就开始. 别等. 等到 2026 年再开始, 你会发现前面已经有一批人在跑, 你追不上了. 真的追不上. 我见过太多这种案例了, 那些 2023 年就开始用 GPT 写 PRD 的 PM, 现在已经是各个团队的中坚; 那些 2024 年才开始用 Cursor 的 PM, 还在焦虑要不要用; 那些 2025 年还在问"AI 到底会不会替代 PM" 的 PM, 老实讲, 我觉得已经晚了半拍.

行动比理解重要. 这话我说给自己听, 也说给你听. 我们这一章是系列最后一章, 整个系列写到这里, 我自己最大的感受是: 这个行业变快了, 但核心没变. PM 干的事还是"在不确定里做判断, 拉着人把事做成". 工具从 PRD 文档变成 agent, 从 Notion 变成 Claude Code, 从飞书会议变成自动纪要, 这些都是形式. 形式会一直变. 不变的是 PM 这个职业最底层的那几件事.

你如果只能从这一章带走一句话, 我希望是这句: 工具会替代你做的事, 不会替代你做的判断. 把时间花在判断上. 这事从今天开始, 不用等下个迭代, 不用等老板批准, 不用等团队对齐. 你的判断水位, 是你自己能控制的唯一变量.

系列到这里结束了. 谢谢你能读到这里. 我自己写这十章也花了不少时间, 回头看, 如果有哪一章能让你少走一点弯路, 那就值得. PM 这个职业不容易, 在 AI 时代更不容易, 但也更有意思. 我们下次再聊.
