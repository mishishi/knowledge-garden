# 04. 分镜: 脚本变画面: 拆镜头 + 视觉化

去年这个时候, 我手上压了 12 条短视频脚本, 全是某品牌方给的文案, 文字本身写得稀烂 (什么"匠心铸就非凡", 你懂的), 但单条 5000 块, 我接了. 第一条我按老办法干, 写脚本 -> 自己脑补画面 -> 直接喂 Sora 跑整段 -> 出来 80% 翻车 -> 改 -> 再跑 -> 两天没睡觉, 出 1 条. 后面 11 条我换了思路, 全部先做分镜图, 再把分镜图当 reference 喂给 AI, 单条平均 3-4 小时, 12 条一周交完. 那一单我挣了 6 万, 但教会我一件事: **分镜不是画给导演看的, 是给 AI 看的**. 这章我想讲清楚这件事.

## 拆镜头这件事, 90% 的人做反了

我最早拆镜头的逻辑是这样的: 拿到脚本, 按"每句话一个镜头"切, 20 秒脚本切 5 个镜头, 每个镜头 4 秒. 听起来很合理对吧? 错得很彻底.

我拿一条"小红书家居博主"的脚本举例 (我自己跑过的, 真实数据). 脚本是这样的: "姐妹们, 我家这个 50 平的小户型, 之前挤得跟鸽子笼似的. 后来我做了一件事, 客厅凭空多出 8 平, 玄关多出 1.5 米, 而且全屋收纳翻倍. 今天就把我踩了 3 套房总结出来的 6 个改造方法全告诉你们." 这种典型的小红书口播钩子文案, 字数大概 180 字, 视频时长控制在 35-45 秒.

我第一版拆了 7 个镜头, 每个对应一段文案. 跑出来是灾难. 为什么? 因为 AI 视频模型 (不管是 Sora 2 / Veo 3 / 可灵还是 Runway) 它根本听不懂"客厅凭空多出 8 平"这种文字指令, 你让它"生成一个客厅变大 8 平的画面", 它会给你一个客厅里突然出现一扇通往另一个客厅的诡异空间门, 直接判你涉黄未遂. 文字描述到画面之间, 中间断层太大了.

后面我学聪明了, 拆镜头按"**画面动作**"拆, 不按"文案段落"拆. 同一条脚本我现在是这样拆的:

- 镜头 1: 真人出镜, 站在 50 平小客厅里, 手势指向整个空间, 表情夸张, 嘴在动 (后面配音)
- 镜头 2: 切到俯拍, 同一个客厅的 3D 平面图动画, 某个区域高亮, 演示"凭空多 8 平"是怎么发生的 (这里是动画镜头, 不是 AI 跑出来的)
- 镜头 3: 切到玄关改造前 vs 改造后对比, 静态图 + 简单平移
- 镜头 4: 真人出镜, 手举 1 到 6 的手指, 嘴动
- 镜头 5: 改造方法 1-2 的前后对比快剪
- 镜头 6: 改造方法 3-4 的前后对比快剪
- 镜头 7: 改造方法 5-6 的前后对比快剪 + 真人结尾

你看到区别了吧? 镜头 1 拍人, 镜头 2 做图, 镜头 3 做对比, 镜头 4 又拍人, 镜头 5/6/7 全是图片快剪. 真正需要 AI 跑出"动态画面"的只有镜头 1 和镜头 4, 而且都是"人物 + 简单背景 + 嘴动"这种最稳定的提示词结构. 其他 5 个镜头要么用图卡 (静态图 + 推拉摇移), 要么用 3D 平面图 (PowerPoint 就能做), 要么用剪映模板.

我后来给自己定了一个规矩: **一个 30-60 秒的短视频, AI 跑视频的镜头不超过 3 个, 其他全用图 / 动画 / 真人素材 / 模板填**. 超过 3 个 AI 镜头, 制作时间翻倍, 翻车率翻 3 倍, 而且一致性会崩 (人物脸忽胖忽瘦, 衣服颜色从粉变紫). 这个数字是我自己跑废了大概 200 条视频总结的, 仅供参考.

## 视觉化: 用 AI 跑分镜图, 不是跑成片

现在聊聊分镜图怎么搞. 很多人搞错了一步: 拿着脚本直接去 Midjourney / GPT-4o / 即梦跑一版图, 然后觉得"分镜就是几张图", 完事. 这是大错特错. **分镜图的核心不是好看, 是给后面的视频生成模型当参考帧 (reference frame) 用的**. 你的图要解决的是: 让 AI 模型知道"这个镜头里有什么, 在哪, 什么风格, 什么光线".

我自己的分镜图流水线是这样的 (2026 年初的版本, 工具一直在换):

第一步, 把拆好的镜头列成一张表, 每行一个镜头, 字段是: 镜头编号 / 画面描述 (中文一句话) / 视觉参考关键词 (英文 3-5 个) / 风格关键词 / 时长 / 备注. 这张表我放在飞书里, 一条脚本对应一个 sheet. 这里有一个关键点: 视觉参考关键词必须是英文, 因为目前所有主流视频生成模型 (Sora 2 / Veo 3.1 / 可灵 2.5 / Runway Gen-4) 对英文 prompt 的理解力比中文好 2-3 个档次, 这是我自己做过对照实验的. 中文 prompt 跑 10 条, 平均 3 条能用; 同样内容翻成英文, 7-8 条能用.

第二步, 把每个镜头的"画面描述"喂给 GPT-4o 或 Claude, 让我帮它扩写成 Midjourney 提示词. 这里有个坑我必须说: 不要直接用 GPT 生成的英文 prompt, 它的 prompt 偏摄影术语 (focal length / depth of field / bokeh), 但视频模型更喜欢"动作 + 主体 + 环境"的结构, 而不是摄影术语结构. 我后来写了个小 prompt 模板, 让 GPT 在生成后强制改成"主体在做什么 + 在哪里 + 周围环境 + 风格"四段式, 可用率从 40% 提到 75%.

第三步, 用 Midjourney v7 或 GPT-4o 的图像生成跑分镜图. 两个我都在用, 场景不一样. Midjourney 跑"有美感 / 有氛围 / 偏电影感"的镜头更好, GPT-4o 跑"逻辑性强 / 构图准 / 人物稳定"的镜头更好. 举个例子, 上面那个小红书脚本的镜头 1, 我用 GPT-4o 跑, 因为它对"亚洲年轻女性, 室内, 手势自然"这种 prompt 还原度更高; 镜头 5/6/7 的对比图我用 Midjourney 跑, 因为更在意"治愈 / 温暖 / 家居杂志感"的氛围. 国产工具里, 即梦 3.0 跑"国风 / 古装 / 东方美学"分镜我用过几次, 比 MJ 强, 但其他场景一般.

第四步, 分镜图跑完不是直接用的, 我会做三件事: (a) 统一画幅 (全部 9:16, 因为 90% 短视频是竖屏); (b) 统一色调, 简单粗暴, 我直接把所有图丢进剪映, 加同一个 LUT 滤镜, 出片风格就齐了; (c) 标注"主体框", 在图上画框标出 AI 视频生成时需要重点保留的人物 / 物品位置, 这个框在 Sora 2 / Veo 3 里可以当 mask 用. 听起来很烦? 习惯之后一条脚本的分镜图处理大概 20-30 分钟.

成本列一下: GPT-4o 图像生成我用的是 ChatGPT Pro 200 美元/月, 不限量 (我一个月平均跑 600-800 张图, 完全够用); Midjourney 我用的是 Standard Plan 30 美元/月, 跑 15 小时 fast GPU, 够用; 即梦我基本只用免费额度, 偶尔开会员. 一个月光工具成本大概 1700 块人民币, 但我做内容一个月流水在 5-8 万, 这点成本不算什么. 如果你是刚起步, 纯用 GPT-4o 免费版 + 即梦免费版也能跑, 速度慢点而已.

## 风格选择: 别追新, 选你的"基因"

这块我得讲讲自己的血泪史. 2024 年 Sora 刚出的时候, 我跟所有人一样, 觉得"电影感 / 大场面 / 一镜到底"就是 AI 视频的未来. 我花了两周, 跑那种"宇航员在火星上骑着机械恐龙"的东西, 发到抖音 / B 站, 播放分别是 200 和 1800. 200 块的成本 (算上时间和电费) 换 1800 播放, 亏爆. 我当时还觉得是运气问题, 又跑了一周, 平均播放没破过 5000.

转折点是我在 B 站看到一条"90 后女生用 AI 给自己做童年老照片动起来"的视频, 那种把静态老照片里的人物"复活"成几秒钟动态的视频, 播放 800 万. 我研究了一下, 这种内容: (a) 情感共鸣强, 童年回忆杀; (b) 技术门槛不高, 可灵 / Runway 都能做; (c) 商业路径清晰, 评论区一堆"帮我做一张, 多少钱". 我第二天就开干, 第一条数据是 12 万播放, 第三条 87 万, 第十五条 320 万. 从那之后我就明白一个道理: **风格不是"我能做什么酷的", 是"我选什么能持续稳定地触达我的目标人群"**.

我现在给学员的建议是这样的, 你自己对照着选. 第一类, 真人出镜 + AI 辅助. 你本来就露脸, AI 帮你做"背景 / 道具 / 场景切换", 这种适合做知识口播 / 带货 / 剧情号. 优势是真人信任度高, 商业化路径最短; 劣势是你要露脸, 而且要会演. 第二类, 图文快剪 + 配音. 全程无 AI 视频生成, 分镜图 + 配音 + 字幕 + 简单动效, 剪映模板一拉就完事. 适合做"盘点 / 知识科普 / 情感语录 / 行业资讯". 这种类型的爆款率高, 制作速度极快 (一条 20 分钟), 一个人一天能出 5-8 条. 我自己的账号矩阵里, 70% 的号是这种类型. 第三类, 全 AI 视频 + 配音 + 剧情. 完全靠 AI 跑出成片, 加上剧情文案和配音. 优势是上限高, 做得好能出圈; 劣势是翻车率高, 一致性差, 制作时间长, 收益不稳定. 新手别碰.

我自己做下来, 数据最稳的是第二类, 收益最高的是第一类 (因为真人出镜的商单价格是图文的 3-5 倍), 投入产出比最差的是第三类 (虽然播放经常爆, 但变现路径极窄, 大部分时候是给平台打工). 这是我跑废了 6 个账号 + 真实接了 100 多单商单之后得出的结论, 不一定适合你, 但可以作为参考.

风格选择上, 还有几个具体建议. 不要追新工具, Sora 2 / Veo 3.1 / 可灵 3.0 哪个都能用, 选你顺手的, 真的差别没那么大. 风格上, 选一个主色调贯穿, 暖色 (米黄 / 棕 / 橙) 适合家居 / 美食 / 情感, 冷色 (蓝 / 灰 / 白) 适合科技 / 职场 / 知识, 高饱和 (红 / 黄 / 紫) 适合潮酷 / 二次元 / 鬼畜, 莫兰迪色适合美妆 / 穿搭 / 慢生活. 别一会儿一个风格, 账号标签会乱, 算法不知道把你推给谁. 这块是我自己踩过的坑, 早期我一个号同时做情感语录和科技测评, 数据怎么都起不来, 后来一分为二, 两个号都起来了.

## 反例: 这些做法是错的

讲几个我亲眼见过 / 自己干过而且赔了钱的反例, 你能避开的就避开. 第一, **直接用 AI 跑成片, 没有分镜**. 这种视频 99% 是"AI 跑一段, 看着不错, 直接发", 数据天花板很低. 因为 AI 模型生成的画面虽然有时候很惊艳, 但缺少叙事节奏, 缺少"钩子", 缺少"信息密度". 观众刷到 3 秒, 没有冲击力, 划走. 好的短视频一定是"画面服务于信息", 不是"画面服务于画面". 第二, **每条视频都换风格**. 上面说过, 但必须再强调一次. 账号做不起来的人, 80% 死在"今天想做这个, 明天想做那个". 你看到一个爆款就模仿一个, 标签全乱, 算法直接把你打入冷宫. 第三, **分镜图跑得过于精美, 视频跟不上**. 见过太多人, 分镜图用 Midjourney 跑了 2 小时, 出来 9 张精修图, 然后视频生成阶段草草用免费工具跑, 出来的视频和分镜图完全两个画风, 整体割裂感极强. 记住一句话: **分镜图是工具, 不是作品**. 它的目的是"让视频模型知道该怎么画", 不是"挂在墙上欣赏". 跑得快比跑得好重要, 跑得风格统一比跑得精细重要.

第四个反例是我自己最痛的一个. 早期我接商单, 客户给脚本, 我按脚本做分镜, 跑视频, 出来客户说"不行, 这个跟我想象的不一样". 来回改 5 版, 最后一版还是用了我第一版. 浪费时间 3 天. 后面我学聪明了, **分镜图出来第一件事是发给客户确认**. 客户点头之后, 再跑视频. 这个改动让我接单效率提升了 60%, 扯皮率降到 10% 以下. 如果你做的是 ToB 商单, 一定要把"分镜确认"作为流程里的一个硬节点, 不要省. ToC 自媒体倒是无所谓, 自己审美自己定.

## 收尾: 今晚就能动手的两件事

第一件事, 找一条你最近看到的爆款短视频, 时长 30-60 秒那种, 按我上面说的方法拆镜头. 不要看文案, 看画面. 一帧一帧暂停, 列出"这个镜头里有什么, 在做什么, 持续几秒, 镜头怎么动". 拆 5 个镜头你就入门了, 拆 10 个你就懂节奏了. 拆完你会发现, 99% 的爆款短视频, 镜头结构就那 6-8 种模板, 来回组合. 这个动作免费, 今晚就能做, 而且我敢说比你看 100 篇"短视频教程"有用.

第二件事, 拿你自己写的一条 30 秒短视频脚本, 拆 5 个镜头, 然后用 GPT-4o 给每个镜头生成 1 张分镜图 (直接说"为这段分镜生成图片"就行, 中文它能听懂). 跑完你就知道, 哪些镜头 AI 跑得好, 哪些跑得烂, 哪些需要你自己画 / 用图库. 这个过程是 30 分钟的事, 但你能立刻感受到"用 AI 跑分镜"和"用 AI 跑成片"的区别. 区别大到你以后再也不想跳过这一步.

下一章我们聊配音 / 配乐 / 音效, 也就是短视频里那 30% 决定"完播率"的听觉部分. 我会讲怎么用 AI 跑人声, 怎么用 Suno / Udio 做背景音乐, 怎么避免"AI 配音一听就是 AI"这个老大难问题. 工具链和成本也都会列清楚, 工程师朋友可以直接抄作业. 链接在文末, 感兴趣可以继续看.
