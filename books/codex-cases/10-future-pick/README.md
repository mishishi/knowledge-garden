# 10. AI Agent 未来 6 个月 + 个人 Pick

写完前 9 章，我自己对 2026 下半年的判断也清晰了。这一章是收尾——讲清楚 3 个核心趋势、3 个独立开发者该抓的机会、3 个我正在押的方向。

## 3 个核心趋势

### 趋势 1：Computer Use 接管遗留系统

**到 2026 年底，30% 企业的"无 API 桌面软件"会被 AI 接管——不是被重写**。

为什么这是趋势？因为重写成本太高。一个 2000 年代的企业 ERP，**重写要 1-2 年 + 500 万**。**Computer Use 接管只要 2-4 周 + $200/月**。

3 个具体场景：

**场景 1：老 ERP 数据搬运**

老 ERP 没 API，新 BI 系统要数据。**传统方案 = 中间库 ETL（开发 1-2 月，维护 6 个月）**。**AI 方案 = Codex Computer Use 每天跑（部署 1 周，0 维护）**。

**场景 2：合规报告生成**

某行业合规报告需要从 5 个老软件导数据 → 填进 Excel → 转 PDF。**人工每月 1 次 × 2 天**。**AI 方案 = 每月自动跑 × 1 小时 review**。

**场景 3：批量录入数据**

需要把 1000 张发票图片录到老财务系统。**人工 = 5 天 × 2 人**。**AI 方案 = Codex + OCR + Computer Use 自动录，1 天跑完**。

**对独立开发者的意义**：**未来 6-12 个月，"AI 操作员" SaaS**会成为大市场。你不需要重写老系统，你**只需要给企业做一个"AI 员工"**，**让 AI 学会用老系统**。

### 趋势 2：Agent Skill 经济成熟

**Agent 技能市场 = 2025 年的 Substack**。

OpenAI 推 6 个行业插件 + 110 skill + 62 集成。Anthropic 推 Claude Code skill 系统。MCP 协议 9700+ servers、1.2 亿月下载。

**这意味着 agent skill 平台**已经成型**。独立开发者的下一个机会**：

- 写一个 Substack 风格的 skill 平台
- 让 agent 开发者上传、订阅、付费
- **市场拐点已经在 2026 上半年发生**

3 个具体机会：

**机会 1：垂直 skill 商店**

通用 agent skill 太宽泛。**垂直行业（医疗 / 法律 / 金融 / 教育）有专属 skill 需求**。做一个"医疗 agent skill 商店"——医院 IT 采购的 ROI 立刻能算。

**机会 2：skill 评分 / 评测**

跟 npm 包一样，**agent skill 也有"哪个好"的问题**。做一个评测平台：跑同一组测试，**给 skill 打分、排序、推荐**。

**机会 3：skill 调试器**

agent 写出来的 skill 经常有问题。**做一个"skill 调试器"——可视化看 skill 怎么调用、为什么失败、怎么优化**。这是 DevTools 类工具，**付费意愿高**。

**对独立开发者的意义**：**未来 6-12 个月，写 agent skill 比写应用更赚钱**。**Substack 早期创作者月入 $10K 的故事会在 agent skill 上重演**。

### 趋势 3：AI 员工 / 工位 vibe coding

**2026 下半年会出现一个新概念——"AI 工位"**。

什么意思？传统公司的组织结构是"人 + 岗位"——一个工程师、一个人事、一个销售。**AI 工位的逻辑是"AI agent + 岗位"**——一个 AI 工程师、AI 人事、AI 销售。

**不是替代人类员工，是重新定义工位**：

- 1 个创业公司 = 1 个创始人 + 1 套 AI 工位 = 10 人小团队产能
- 1 个中型公司 = 1 个 leader + 20 个 AI 工位 = 100 人公司产能

**这个概念的关键不是"AI 多聪明"，而是"AI 工位多稳定"**。

3 个早期信号：

**信号 1：AI 工位 SaaS 出现**

2026 上半年已经有 5+ 公司做"AI 员工平台"——提供预配置好的 AI 销售 / AI 客服 / AI 数据分析师。**月费 $200-2000**。

**信号 2：工位 vibe coding 工具出现**

新工具让"配置 AI 工位"像"配置 web 服务"——可视化拖拽、设置任务边界、调整权限。**未来 6 个月这类工具会爆发**。

**信号 3：投资人押注 AI 工位赛道**

2026 年 5 月 CB Insights 报告：5 个新晋独角兽里有 1 个做 AI 员工。**Capgemini 预测 82% 企业到 2026 年集成 AI Agent**。

**对独立开发者的意义**：**你未来要做的不是"卖给公司一个产品"，而是"卖给公司一个 AI 工位"**——**后者单客单价高 10 倍**（$500-5000/月 vs $50-200/月）。

## 3 个我正在押的方向

我自己 6-12 个月的 3 个押注：

### 押注 1：AI 操作员（Computer Use 接管遗留系统）

**目标**：给中小企业的"老软件"做 AI 操作员。

**具体形态**：

- 桌面 Agent（Codex Computer Use 基础）
- 任务库（每个任务 = 一个"AI 操作员"角色）
- 计费（按任务 / 按时间）

**商业模型**：

- 基础版：$99/月（1 个工位 + 5 个任务）
- 团队版：$499/月（5 个工位 + 30 个任务）
- 企业版：$2000+/月（定制）

**为什么押**：

- 需求真实：每个中小企业都有"老系统数据搬运"的痛
- 壁垒不高但有先发：Computer Use 平台未定，**谁先做谁占位**
- 单客价高：传统 RPA 客户付费意愿 $500-2000/月

### 押注 2：Agent Skill 商店

**目标**：做一个"agent skill 集市"——开发者上传 skill，平台抽成。

**具体形态**：

- Skill 上传 + 评分 + 付费
- Agent 端 API（让 agent 直接调 skill）
- 创作者后台（看使用量 / 收益）

**商业模型**：

- 抽成 20%（类似 Apple Store）
- 创作者 80%

**为什么押**：

- Substack 早期是"邮箱订阅"市场，**agent skill 是"agent 工具调用"市场**——**逻辑相同、规模更大**
- MCP 协议成熟后，**跨平台分发成本低**
- 头部创作者月入 $10K+ 完全可期

### 押注 3：Vibe Coding 课程 / 工具

**目标**：教"非程序员用 AI 写代码"。

**具体形态**：

- 课程：5 节课教产品经理 / 运营用 Codex 做内部工具
- 工具：vibe coding starter kit（模板 + skill 库 + 最佳实践）
- 社区：Discord 群（$29/月订阅）

**商业模型**：

- 课程 $99-499
- 工具 $49/项目
- 社区 $29/月

**为什么押**：

- Vibe coding 是 2026 最大的"非程序员"AI 应用市场
- **"写代码" 的门槛大幅下降**，**"用 AI 写代码" 的认知还在早期**
- **教人"会 AI 写代码" 比教人"会写代码" ROI 高**

## 3 个值得学的 2026 工具 / 项目

如果你现在刚开始 agent 实战，**这 3 个值得花时间学**：

### 1. CodexLoop（强烈推荐）

- 仓库：github.com/kunkunzhishan/codexloop
- 解决的问题：AI 长任务偷懒
- 学习时间：1 天读代码 + 1 周实操
- 适合：所有用 Codex / Claude Code 做长任务的人

**为什么必学**：

- "AI 偷懒" 是 2026 仍未解的通用问题
- CodexLoop 的 4 个机制（Checklist / Review / Deferred Ideas / Audit Logs）**适用于所有 agent**
- 开源 + 本地 = **可商用**（做付费版）

### 2. AIGrader（架构参考）

- 仓库：github.com/xiaodangjia/AIGrader
- 解决的问题：1 人半天全栈项目
- 学习时间：2-3 天看完整项目
- 适合：所有想用 AI 提速独立开发的独立开发者

**为什么必学**：

- 验证了"6 stream 并行 + 3 层约束"的可复用模式
- 是"独立开发者 = 1 个公司" 的最早实证
- 给了独立开发者"半天出 MVP" 的具体路径

### 3. 飞书 CLI（办公自动化）

- 仓库：github.com/larksuite/cli
- 解决的问题：Agent 接入办公系统
- 学习时间：1-2 天实操
- 适合：所有用飞书 / 想做办公自动化的中文独立开发者

**为什么必学**：

- 中文办公自动化最大市场的入口
- 10.7k star = 已被验证可用
- 官方把 AI agent 写入 roadmap，**未来 1 年功能爆发**

## 1 个我建议避开的陷阱

**避开："做一个通用的 agent OS"**。

很多独立开发者想"做 agent 时代的操作系统"——通用平台、通用框架、通用工具。

**这个方向对独立开发者来说太重**：

- 通用 agent OS = 跟 OpenAI / Anthropic / LangChain 竞争
- **单家公司** 投入 $50M-$200M
- **独立开发者** 不可能赢

**我建议专注"垂直"**：

- 垂直行业（医疗 / 法律 / 教育 / 金融）
- 垂直场景（Computer Use 接管 / Skill 商店 / 工位 vibe coding）
- 垂直用户（独立开发者 / 中小企业 / 大企业部门）

**Substack vs Twitter 的故事会重演**——**垂直 skill 商店赢过通用 agent 平台**。

## 1 句话总结

**未来 6 个月，独立开发者的 ROI 最高的方向是"AI 操作员 + Agent Skill 商店 + Vibe Coding 工具"三个赛道的任意一个**。

不是"做 agent 应用"，是"做 agent 时代的 skill 卖家"。

不是"做 AI 产品"，是"做 AI 工位"。

**Agent 时代的产品经理 = 独立开发者的新身份**。**不是"写代码的人"，是"设计 agent 协作的人"**。
