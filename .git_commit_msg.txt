feat(kb-history): Q&A 历史回看 + 时间显示 + 缓存结果

UX #2: Q&A 搜索历史从纯文字升级, 点行可回看上次结果

- 存 {query, html, ts} 对象 (老 string[] 格式兼容, 第一次访问自动迁移)
- 搜索后自动存 results.innerHTML 到 history
- 点历史行 → 直接渲染缓存 HTML, 顶部 banner 提示从历史恢复
- banner 有 "重新搜索" 按钮 (输入变化或想更新结果时用)
- 显示相对时间 (刚刚 / 5 分钟前 / 2 天前 / 日期)
- 状态提示: 已存结果 vs 再次搜索 (缓存缺失时)
- 老数据平滑迁移

UX: 用户搜过 RAG → 2 天后再想看, 打开 Q&A modal 一眼回看, 不必重新跑 TF-IDF
